Masz gotową stronę HTML. Wygląda świetnie. Ale gdy klikasz przycisk — nic. Gdy wpisujesz tekst — nic. Strona jest jak zdjęcie: możesz patrzeć, ale nie możesz dotknąć.

Żeby strona reagowała na użytkownika, JavaScript musi mieć dostęp do jej elementów. Ten dostęp daje <Term word="DOM">DOM</Term> — Document Object Model. To drzewo obiektów, które przeglądarka buduje z Twojego HTML. Każdy <Term word="znacznik">znacznik</Term> staje się obiektem, każdy <Term word="atrybut">atrybut</Term> staje się właściwością. JavaScript może to drzewo czytać, zmieniać i reagować na to, co robi użytkownik.

## Jak HTML staje się drzewem

Gdy przeglądarka wczyta HTML, nie widzi tekstu — widzi strukturę. Każdy element ma swojego rodzica, swoje dzieci i swoich sąsiadów.

```html
<html>
  <body>
    <h1 id="tytul">Cześć</h1>
    <p class="opis">Witaj na stronie</p>
  </body>
</html>
```

W pamięci przeglądarki to wygląda tak:

```
document
└── html
    └── body
        ├── h1#tytul
        │   └── "Cześć"           ← węzeł tekstowy
        └── p.opis
            └── "Witaj na stronie" ← węzeł tekstowy
```

JavaScript ma do tego drzewa pełen dostęp przez globalny obiekt `document`. To punkt wejścia do całej strony.

## Wybieranie elementów — zanim cokolwiek zmienisz

Żeby coś zmienić, musisz to najpierw znaleźć. Masz do tego dwie główne metody.

### `querySelector` — pierwszy pasujący element

```js
const tytul = document.querySelector("h1");
const tytul2 = document.querySelector("#tytul");          // po id
const opis = document.querySelector(".opis");             // po klasie
const input = document.querySelector('input[type="email"]'); // selektor CSS
const button = document.querySelector("form button");     // zagnieżdżony
```

Używasz dokładnie tych samych selektorów co w CSS. Jeśli element nie istnieje, dostajesz `null` — warto to sprawdzić przed użyciem.

### `querySelectorAll` — wszystkie pasujące

```js
const wszystkieParagrafy = document.querySelectorAll("p");
const inputy = document.querySelectorAll('input[type="text"]');
```

Zwraca NodeList — możesz po niej iterować `forEach` albo klasycznym `for`.

### Starsze metody — znaj, ale nie preferuj

```js
document.getElementById("tytul");
document.getElementsByClassName("opis");     // HTMLCollection
document.getElementsByTagName("p");
```

Działają, ale `querySelector*` jest czytelniejszy i bardziej elastyczny. Zobaczysz te starsze w cudzym kodzie — rozumiej je, ale sam pisz nowocześnie.

<Callout type="tip" title="querySelector to CSS w JS">
Jeśli umiesz napisać selektor CSS, umiesz też wybrać element przez querySelector. To ta sama składnia — nie musisz uczyć się niczego nowego.
</Callout>

## Zmiana treści — co widzisz na stronie

Gdy już masz element, możesz zmienić to, co wyświetla.

### `textContent` — bezpieczny tekst

```js
const tytul = document.querySelector("h1");
tytul.textContent = "Nowy tytuł";
console.log(tytul.textContent);   // "Nowy tytuł"
```

`textContent` ustawia i pobiera czysty tekst. Jeśli wstawisz tu coś, co wygląda jak HTML, zostanie wyświetlone dosłownie jako tekst — nie jako znaczniki. To czyni go **bezpiecznym przed <Term word="XSS">XSS</Term>**.

### `innerHTML` — wstawia prawdziwy HTML

```js
const kontener = document.querySelector("#app");
kontener.innerHTML = "<strong>Ważne!</strong> Nowa treść.";
```

`innerHTML` parsuje string jako HTML i wstawia go do elementu. Potężne, ale niebezpieczne — jeśli wstawisz tu dane od użytkownika bez oczyszczenia, otwierasz drzwi dla ataku <Term word="XSS">XSS</Term>. Używaj tylko z danymi, którym ufasz.

### `value` — wartość pola formularza

```js
const input = document.querySelector("#email");
console.log(input.value);          // pobierz to, co wpisał użytkownik
input.value = "nowy@email.com";   // ustaw wartość programowo
```

## Zmiana klas i atrybutów

Często zamiast zmieniać treść, chcesz zmienić wygląd — a to robisz przez klasy CSS.

### `classList` — operacje na klasach

```js
const box = document.querySelector(".box");

box.classList.add("aktywna");            // dodaj klasę
box.classList.remove("aktywna");         // usuń klasę
box.classList.toggle("aktywna");         // przełącz (dodaj jeśli nie ma, usuń jeśli jest)
box.classList.contains("aktywna");       // sprawdź, czy ma
box.classList.replace("stara", "nowa");  // zamień jedną na drugą
```

`toggle` to złoto — jedną linią obsługujesz menu hamburgerowe, modal, przełącznik motywu. Nie musisz sprawdzać, czy klasa jest, czy jej nie ma.

### `setAttribute` / `getAttribute` / `removeAttribute`

```js
const link = document.querySelector("a");
link.setAttribute("href", "https://example.com");
link.setAttribute("target", "_blank");
link.setAttribute("rel", "noopener");
console.log(link.getAttribute("href"));
link.removeAttribute("target");
```

<Callout type="tip" title="Kiedy setAttribute, kiedy właściwość DOM">
- Atrybuty niestandardowe (`data-*`) — `setAttribute("data-id", 5)`.
- Standardowe właściwości (`href`, `src`, `value`, `checked`) — bezpośrednio: `link.href = "..."`.
- Właściwości lepiej oddają aktualny stan: `input.checked` zwraca boolean, `getAttribute("checked")` zwraca string.
</Callout>

## Zdarzenia — reagowanie na to, co robi użytkownik

To serce interaktywności. Zdarzenie to sygnał: „coś się stało". Kliknięcie, wciśnięcie klawisza, wysłanie formularza, zmiana wartości pola — każde z nich to zdarzenie, na które możesz zareagować.

### `addEventListener` — nasłuchuj i reaguj

```js
const button = document.querySelector("#klik");

button.addEventListener("click", (event) => {
    console.log("Kliknięto!", event);
});
```

Trzy elementy: element, który nasłuchujesz (`button`), nazwa zdarzenia (`"click"`) i funkcja, która się wykona gdy zdarzenie zajdzie. Obiekt `event` niesie szczegóły: typ zdarzenia, który element go wywołał (`event.target`), pozycja kursora, wciśnięte klawisze.

### Najczęstsze zdarzenia

| Zdarzenie | Kiedy |
|-----------|-------|
| `click` | Kliknięcie myszką / tap |
| `dblclick` | Podwójne kliknięcie |
| `mouseenter` / `mouseleave` | Kursor wjechał / wyjechał |
| `keydown` / `keyup` | Klawisz wciśnięty / puszczony |
| `submit` | Formularz wysłany |
| `input` | Wartość pola zmieniona (każdy znak) |
| `change` | Wartość pola zmieniona (po opuszczeniu pola) |
| `focus` / `blur` | Pole zyskało / straciło fokus |
| `scroll` | Strona lub element przescrollowany |

### Przykład — licznik kliknięć

```js
let count = 0;
const button = document.querySelector("#klik");
const display = document.querySelector("#licznik");

button.addEventListener("click", () => {
    count++;
    display.textContent = count;
});
```

Każde kliknięcie zwiększa licznik i natychmiast aktualizuje to, co widzi użytkownik. Żadnego przeładowania strony.

### Przykład — przełącznik motywu ciemny/jasny

```js
const themeBtn = document.querySelector("#theme");
themeBtn.addEventListener("click", () => {
    document.body.classList.toggle("dark");
    localStorage.setItem("theme", document.body.classList.contains("dark") ? "dark" : "light");
});
```

`toggle` na klasie `dark` przełącza motyw. `localStorage` zapamiętuje wybór użytkownika między odświeżeniami strony.

## Tworzenie i usuwanie elementów

Nie musisz pisać HTML z góry — możesz tworzyć elementy dynamicznie, gdy są potrzebne.

### `createElement` + `appendChild`

```js
// Stwórz nowy element
const li = document.createElement("li");
li.textContent = "Nowy element listy";
li.classList.add("item");

// Dodaj do istniejącego elementu
const lista = document.querySelector("#lista");
lista.appendChild(li);    // na koniec
lista.prepend(li);        // na początek

// Wstaw przed konkretnym elementem
const specjalny = document.querySelector("#specjalny");
lista.insertBefore(li, specjalny);
```

### `remove` — usuń element

```js
const doUsuniecia = document.querySelector("#tymczasowy");
doUsuniecia.remove();
```

### `cloneNode` — skopiuj element

```js
const oryginal = document.querySelector(".karta");
const kopia = oryginal.cloneNode(true);  // true = kopiuj też dzieci
document.querySelector("#galeria").appendChild(kopia);
```

## Event delegation — jeden listener zamiast wielu

Wyobraź sobie listę zadań z setką pozycji. Możesz dodać listener do każdego przycisku „usuń" osobno — albo dodać jeden listener do rodzica i sprawdzić, który element wywołał zdarzenie.

```html
<ul id="todo-list">
    <li>Kup mleko <button class="delete">×</button></li>
    <li>Nakarm kota <button class="delete">×</button></li>
</ul>
```

```js
const lista = document.querySelector("#todo-list");

lista.addEventListener("click", (e) => {
    if (e.target.classList.contains("delete")) {
        e.target.parentElement.remove();
    }
});
```

`e.target` to dokładnie ten element, który kliknął użytkownik. Sprawdzasz, czy ma klasę `delete` — jeśli tak, usuwasz jego rodzica (cały `<li>`).

Dwie zalety: mniej kodu i obsługa elementów dodanych dynamicznie. Listener na rodzicu działa też dla `<li>`, które dopiero za chwilę stworzysz przez `appendChild`.

<Callout type="info" title="Event delegation to wzorzec, nie trick">
To standardowy sposób obsługi dynamicznych list w JavaScript. Zobaczysz go w każdym projekcie — naucz się go rozumieć i pisać z pamięci.
</Callout>

## Zapobieganie domyślnym akcjom

Przeglądarka ma swoje domyślne zachowania: formularz wysyła dane i przeładowuje stronę, link przenosi na inny adres. Czasem chcesz to przejąć i zrobić coś własnego.

```js
// Formularz nie wysyła się, dopóki nie zwalidujesz
form.addEventListener("submit", (e) => {
    e.preventDefault();
    if (waliduj(form)) {
        form.submit();
    }
});

// Link nie otwiera się
link.addEventListener("click", (e) => {
    e.preventDefault();
    // zrób coś innego
});
```

`e.preventDefault()` anuluje domyślną akcję przeglądarki. Bez niego formularz wysłałby się i strona by się przeładowała — straciłbyś kontrolę.

## `DOMContentLoaded` — czekaj, aż strona będzie gotowa

Skrypt z `defer` wykonuje się po sparsowaniu HTML — to bezpieczne. Ale jeśli z jakiegoś powodu nie używasz `defer`, Twój kod może się wykonać, zanim przeglądarka zbuduje drzewo <Term word="DOM">DOM</Term>. Wtedy `querySelector` zwróci `null` i nic nie zadziała.

```js
document.addEventListener("DOMContentLoaded", () => {
    const btn = document.querySelector("#klik");
    btn.addEventListener("click", () => {
        console.log("Gotowe!");
    });
});
```

Cały kod wewnątrz callbacku wykona się dopiero, gdy HTML jest w pełni sparsowany. Alternatywa: po prostu zawsze używaj `defer` na znaczniku `<script>` — efekt ten sam, bez dodatkowego kodu.

<Callout type="exam">
Na egzaminie INF.03 z DOM sprawdzają: poprawne użycie `querySelector` z selektorem CSS, zmianę `textContent` (nie `innerHTML` do zwykłego tekstu), podpięcie zdarzenia przez `addEventListener` (nie `onclick` w HTML), oraz `e.preventDefault()` przy obsłudze formularza. Pamiętaj też o różnicy między `textContent` (bezpieczny) a `innerHTML` (ryzykowny przy danych od użytkownika).
</Callout>

## Podsumowanie

`querySelector` / `querySelectorAll` do wybierania — te same selektory co w CSS. `textContent` do tekstu (bezpieczny przed <Term word="XSS">XSS</Term>), `innerHTML` do HTML (ostrożnie z danymi od użytkownika). `classList` do klas, `setAttribute` do atrybutów niestandardowych. `addEventListener("click", fn)` do zdarzeń — nie `onclick` w HTML. `createElement` + `appendChild` do tworzenia, `.remove()` do usuwania. Event delegation na rodzicu = jeden listener zamiast wielu, działa też dla dynamicznie dodanych elementów. `e.preventDefault()` anuluje domyślną akcję przeglądarki. `defer` na `<script>` gwarantuje, że drzewo jest gotowe — bez potrzeby `DOMContentLoaded`.

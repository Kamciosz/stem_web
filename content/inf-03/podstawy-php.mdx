Otwierasz w przeglądarce stronę sklepu. Widzisz swoje imię w prawym górnym rogu, liczbę produktów w koszyku, ceny przeliczone na złotówki. Nikt nie wpisał tego ręcznie w HTML — to <Term word="PHP">PHP</Term> wygenerował tę stronę chwilę wcześniej, na serwerze, specjalnie dla Ciebie. HTML jest jak wydrukowana ulotka: zawsze taka sama. PHP jest jak sprzedawca, który za każdym razem mówi coś innego, zależnie od tego, kto wszedł.

W tej lekcji poznasz fundament: jak PHP działa, jak pisać zmienne i warunki, jak odbierać dane z formularza i — to najważniejsze — jak nie dać się zhakować już na starcie.

## Czym jest PHP i gdzie się wykonuje

Najpierw zrozum jedną rzecz, bo z niej wynika cała reszta. PHP to język <Term word="server-side">server-side</Term> — wykonuje się na <Term word="serwer">serwerze</Term>, nie w przeglądarce. Kiedy ktoś otwiera stronę `.php`, serwer uruchamia skrypt, a do przeglądarki wysyła **gotowy HTML**. Użytkownik nigdy nie widzi Twojego kodu PHP — widzi tylko wynik.

PHP (PHP: Hypertext Preprocessor) potrafi:

- Wykonać się po stronie serwera, zanim cokolwiek dotrze do użytkownika.
- Generować HTML dynamicznie — inny dla każdego.
- Łączyć się z bazami danych, plikami, <Term word="API">API</Term>.
- Wbudowywać się wprost w HTML za pomocą znaczników `<?php ... ?>`.

Spójrz, jak PHP miesza się z HTML-em. Wszystko poza `<?php ?>` to zwykły HTML, a w środku dzieje się magia:

```php
<!DOCTYPE html>
<html lang="pl">
<head>
    <meta charset="UTF-8">
    <title>PHP Demo</title>
</head>
<body>
    <h1>Witaj, <?= htmlspecialchars($imie ?? "Gość") ?>!</h1>
    <p>Aktualny czas: <?= date("Y-m-d H:i:s") ?></p>
</body>
</html>
```

Zwróć uwagę na `$imie ?? "Gość"` — jeśli zmienna `$imie` nie istnieje, użyj "Gość". To operator zwany null coalescing, będziesz go używać bez przerwy.

<Callout type="tip" title="Środowisko lokalne">
Do testowania PHP potrzebujesz serwera — sam plik otwarty z dysku nie zadziała. Na macOS/Windows najłatwiej **XAMPP** (Apache + MySQL + PHP). Pliki umieszczaj w `htdocs/`, strony otwieraj przez `http://localhost/`. Bez serwera przeglądarka pokaże Ci surowy kod zamiast wyniku.
</Callout>

## Składnia — to, od czego zaczynasz

### Znaczniki PHP

Kod PHP zawsze żyje między tymi znacznikami:

```php
<?php
// Kod PHP
?>
```

Jest też skrót do wypisywania: `<?= $zmienna ?>` robi dokładnie to samo co `<?php echo $zmienna ?>`. Używaj go, gdy chcesz tylko wstawić wartość do HTML — jest krótszy i czytelniejszy.

### Komentarze

```php
// jednolinijkowy (C++)
# jednolinijkowy (shell)
/* wielolinijkowy (C) */
```

### Średnik

Każda instrukcja kończy się średnikiem. To nie jest opcja — to wymóg. Zapomnisz średnika i dostaniesz biały ekran (parse error). Gdy coś nie działa, a ekran jest pusty, najpierw szukaj brakującego `;`.

## Zmienne

W PHP każda zmienna zaczyna się od `$`. Bez wyjątków:

```php
$imie = "Anna";          // string
$wiek = 16;              // int
$średnia = 4.5;          // float
$aktywny = true;          // bool
$pusta = null;            // null (jak undefined)

// Tablica
$kolory = ["czerwony", "zielony", "niebieski"];
$osoba = ["imie" => "Anna", "wiek" => 16];
```

Teraz uwaga na pułapkę. PHP jest **słabo typowany** — ta sama zmienna może raz trzymać liczbę, raz tekst. Brzmi wygodnie, ale prowadzi do trudnych błędów. Dlatego w funkcjach deklaruj typy — kod staje się czytelniejszy i bezpieczniejszy:

```php
function przywitaj(string $imie, int $wiek): string {
    return "Cześć, mam na imię $imie i mam $wiek lat.";
}
```

Gdy wartość nigdy się nie zmienia (numer wersji, nazwa aplikacji), użyj stałej:

```php
define("WERSJA", "1.0.0");
const NAZWA_APLIKACJI = "Koło STEM";
```

## Operatory

Większość znasz z innych języków, ale jeden zestaw musisz zapamiętać szczególnie dobrze — porównania:

```php
$a + $b;          // dodawanie
$a - $b;          // odejmowanie
$a * $b;          // mnożenie
$a / $b;          // dzielenie
$a % $b;          // modulo (reszta)
$a ** $b;         // potęga

$a == $b;         // równe (z konwersją typów)
$a === $b;        // identyczne (typ + wartość) — PREFEROWANE
$a != $b;         // różne
$a !== $b;        // nieidentyczne

$a && $b;         // AND
$a || $b;         // OR
!$a;              // NOT
```

<Callout type="warning" title="=== vs ==">
W PHP `==` potrafi zaskoczyć: `"0" == false` zwraca `true`, bo PHP konwertuje typy w tle i uznaje oba za "fałszywe". To źródło prawdziwych dziur bezpieczeństwa — np. przy sprawdzaniu tokenów. **Zawsze używaj `===`** do ścisłego porównania. Trzy znaki równości to Twój domyślny wybór.
</Callout>

## Warunki

```php
if ($wiek >= 18) {
    echo "Pełnoletni";
} elseif ($wiek >= 13) {
    echo "Nastolatek";
} else {
    echo "Dziecko";
}

// Trójargumentowy
$kategoria = ($wiek >= 18) ? "dorosły" : "dziecko";

// Match (PHP 8+)
$komunikat = match (true) {
    $wiek >= 18 => "Pełnoletni",
    $wiek >= 13 => "Nastolatek",
    default     => "Dziecko",
};
```

`match` to nowszy, czystszy sposób na wiele warunków — warto go znać, bo nie wymaga `break` i porównuje przez `===`.

## Pętle

Pętle robią to samo, co w każdym języku — powtarzają kod. W PHP najczęściej sięgniesz po `foreach`, bo świetnie chodzi po tablicach:

```php
// for
for ($i = 0; $i < 5; $i++) {
    echo "Iteracja $i";
}

// while
$count = 0;
while ($count < 5) {
    echo $count++;
}

// foreach — po tablicy
$owoce = ["jabłko", "banan", "gruszka"];
foreach ($owoce as $owoc) {
    echo $owoc;
}

// foreach — klucz + wartość
$osoba = ["imie" => "Anna", "wiek" => 16];
foreach ($osoba as $klucz => $wartosc) {
    echo "$klucz: $wartosc\n";
}
```

## Tablice

Tablica to pojemnik na wiele wartości. PHP ma dwa rodzaje: indeksowaną (numerowaną od zera) i asocjacyjną (klucz → wartość). Tej drugiej będziesz używać przy danych z bazy i formularzy, więc poznaj ją dobrze:

```php
// Indeksowana
$kolory = ["czerwony", "zielony", "niebieski"];
echo $kolory[0];                    // "czerwony"
array_push($kolory, "żółty");       // dodaj na koniec

// Asocjacyjna (klucz => wartość)
$osoba = ["imie" => "Anna", "wiek" => 16, "email" => "anna@example.com"];
echo $osoba["imie"];                // "Anna"

// Iteracja
foreach ($osoba as $klucz => $wartosc) {
    echo "$klucz: $wartosc\n";
}

// Sprawdzenie czy klucz istnieje
if (array_key_exists("email", $osoba)) {
    echo $osoba["email"];
}

// Sprawdzenie czy wartość istnieje
if (in_array("Anna", $osoba)) {
    echo "Anna jest w tablicy";
}
```

## Formularze — tu zaczyna się prawdziwa robota

Do tej pory dane wpisywałeś sam w kodzie. Teraz przychodzą one od użytkownika — przez formularz. To moment, w którym PHP staje się interaktywny.

Najpierw formularz po stronie HTML (metoda <Term word="POST">POST</Term>, bo wysyłamy hasło):

```html
<form action="/login.php" method="post">
    <label for="email">E-mail</label>
    <input id="email" name="email" type="email" required>

    <label for="pass">Hasło</label>
    <input id="pass" name="password" type="password" required>

    <button type="submit">Zaloguj</button>
</form>
```

A teraz odbiór danych w PHP. Zauważ, że sięgamy po dane przez `$_POST["nazwa_pola"]` — gdzie "nazwa_pola" to dokładnie ten `name` z formularza:

```php
<?php
if ($_SERVER["REQUEST_METHOD"] === "POST") {
    $email = $_POST["email"] ?? "";
    $haslo = $_POST["password"] ?? "";

    if (empty($email) || empty($haslo)) {
        echo "Uzupełnij wszystkie pola.";
    } else {
        // Logika logowania
        echo "Próba logowania: $email";
    }
}
```

### `$_GET` vs `$_POST`

To pytanie pada na egzaminie i w każdej rozmowie o pracę. Zapamiętaj różnicę:

- **`$_GET`** — dane z URL (`?klucz=wartość`). Widoczne w pasku adresu. Metoda <Term word="GET">GET</Term> służy do wyszukiwarek, filtrów, sortowania.
- **`$_POST`** — dane w body żądania. Niewidoczne w URL. Do formularzy z danymi wrażliwymi: logowanie, rejestracja.
- **`$_REQUEST`** — połączenie obu. Mieszane i nieprzewidywalne — raczej unikaj.

```php
// URL: /szukaj.php?q=php
$query = $_GET["q"] ?? "brak";
echo "Szukasz: $query";
```

## Bezpieczeństwo — to jest punktowane na egzaminie

Teraz najważniejsza część całej lekcji. Zapamiętaj złotą zasadę backendu: **nigdy nie ufaj danym od użytkownika**. Użytkownik to nie zawsze grzeczny gość — czasem to ktoś, kto celowo wpisuje w pole imienia kawałek kodu, żeby przejąć Twoją stronę. Twoim zadaniem jest go zatrzymać.

### XSS — gdy użytkownik wstrzykuje kod do Twojej strony

<Term word="XSS">XSS</Term> (Cross-Site Scripting) to atak, w którym ktoś wpisuje w pole formularza nie tekst, a kod JavaScript. Jeśli wyświetlisz to bez zabezpieczenia, ten kod **wykona się w przeglądarce każdego, kto wejdzie na stronę**. Lekarstwo jest proste i ma jedno nazwisko: `htmlspecialchars()`.

```php
$imie = $_GET["name"] ?? "Gość";
// ❌ Niebezpieczne — user może wstrzyknąć <script>
echo "Witaj, $imie";

// ✅ Bezpieczne — encje HTML
echo "Witaj, " . htmlspecialchars($imie, ENT_QUOTES, "UTF-8");

// Skrót:
// echo "Witaj, " . htmlspecialchars($imie);
```

`htmlspecialchars()` zamienia `<` na `&lt;`, `>` na `&gt;`, `"` na `&quot;`. Dzięki temu, gdy ktoś wpisze `<script>alert('XSS')</script>`, przeglądarka **wyświetli to jako zwykły tekst** zamiast wykonać. Bez tej funkcji haker uruchomiłby swój JavaScript u każdego odwiedzającego.

<Callout type="exam">
Na egzaminie INF.03 oceniający szukają `htmlspecialchars()` przy KAŻDYM wyświetlaniu danych pochodzących od użytkownika (z `$_GET`, `$_POST`, z bazy). Pominięcie go to nie drobiazg — to błąd bezpieczeństwa, za który tracisz punkty. Wpisz to sobie w nawyk: dane od usera na ekran = `htmlspecialchars()`.
</Callout>

### Walidacja danych

<Term word="walidacja">Walidacja</Term> to sprawdzanie, czy dane mają sens, zanim cokolwiek z nimi zrobisz. PHP daje gotowe narzędzia:

```php
function walidujEmail(string $email): bool {
    return filter_var($email, FILTER_VALIDATE_EMAIL) !== false;
}

function walidujWiek(mixed $wiek): ?int {
    if (!is_numeric($wiek)) return null;
    $int = (int)$wiek;
    if ($int < 0 || $int > 150) return null;
    return $int;
}
```

### SQL injection — krótka zapowiedź

Drugi wielki atak to <Term word="SQL injection">SQL injection</Term> — wstrzyknięcie złośliwego kodu <Term word="SQL">SQL</Term> przez dane od usera. **Nigdy** nie sklejaj zapytania SQL ze zmiennej od użytkownika. Obroną są <Term word="prepared statements">prepared statements</Term>, którym poświęcona jest osobna lekcja (`php-mysql`). Na razie zapamiętaj samo ostrzeżenie — wrócimy do tego z konkretnym kodem.

## Funkcje wbudowane — Twój codzienny zestaw

PHP ma setki gotowych funkcji. Tych użyjesz najczęściej:

```php
// Stringi
strlen("Ala ma kota");                  // 11
strtoupper("cześć");                    // "CZEŚĆ"
strtolower("CZEŚĆ");                    // "cześć"
str_replace("kota", "psa", "Ala ma kota");  // "Ala ma psa"
trim("  tekst  ");                      // "tekst"
substr("Ala ma kota", 0, 3);            // "Ala"
str_contains("Ala ma kota", "kot");     // true (PHP 8+)

// Tablice
count([1, 2, 3]);                       // 3
array_merge([1, 2], [3, 4]);            // [1, 2, 3, 4]
array_filter([1, 2, 3, 4], fn($x) => $x > 2);  // [3, 4]
array_map(fn($x) => $x * 2, [1, 2, 3]); // [2, 4, 6]
sort([3, 1, 2]);                        // sortuje w miejscu

// Daty
date("Y-m-d");                          // "2026-06-01"
date("Y-m-d H:i:s");                    // "2026-06-01 14:30:00"
time();                                 // Unix timestamp

// JSON
json_encode(["imie" => "Anna"]);        // '{"imie":"Anna"}'
json_decode('{"imie":"Anna"}', true);   // ["imie" => "Anna"]

// Bezpieczeństwo
password_hash("haslo", PASSWORD_DEFAULT);
password_verify("haslo", $hash);
htmlspecialchars("<script>");
filter_var("anna@example.com", FILTER_VALIDATE_EMAIL);
```

Zwróć uwagę na dwie ostatnie z sekcji bezpieczeństwa: `password_hash()` zamienia hasło w nieczytelny skrót przed zapisem do bazy, a `password_verify()` sprawdza, czy wpisane hasło pasuje do tego skrótu. **Nigdy nie zapisuj hasła jako zwykłego tekstu** — zawsze przez `password_hash`.

## Dołączanie plików

Po co kopiować ten sam nagłówek do dziesięciu plików, skoro można go napisać raz i dołączać? Do tego służą `include` i `require`:

```php
// Wstawia i wykonuje plik. UWAGA na zmienne!
include "header.php";        // błąd → warning, skrypt działa dalej
require "header.php";        // błąd → fatal error
include_once "config.php";   // tylko raz (zapobiega ponownemu)
require_once "db.php";       // połączenie z bazą raz
```

Różnica jest prosta: `require` przy błędzie zatrzymuje cały skrypt (użyj go do rzeczy krytycznych, np. połączenia z bazą), `include` tylko ostrzega. Wersje `_once` gwarantują, że plik wczyta się najwyżej raz — kluczowe przy połączeniu z bazą.

Typowy układ projektu wygląda tak:

```
/index.php
/header.php    ← górna część HTML + nawigacja
/footer.php    ← stopka
/db.php        ← połączenie z bazą (require_once)
```

```php
<?php require_once "header.php"; ?>

<h1>Witaj</h1>
<p>Treść strony</p>

<?php require_once "footer.php"; ?>
```

## Podsumowanie

PHP to język <Term word="server-side">server-side</Term> ze składnią podobną do C — wykonuje się na serwerze, a użytkownik dostaje gotowy HTML. Zmienne zaczynają się od `$`. PHP jest słabo typowany, więc deklaruj typy w funkcjach i porównuj przez `===`, nie `==`. Dane z formularzy odbierasz przez `$_GET` (URL, wyszukiwanie) i `$_POST` (body, dane wrażliwe). I dwie zasady, które ratują Ci skórę na egzaminie i w prawdziwej pracy: `htmlspecialchars()` przy KAŻDYM wyświetlaniu danych od użytkownika (ochrona przed XSS) oraz `password_hash` / `password_verify` do haseł. Do bezpiecznego SQL-a służą prepared statements — to temat następnej lekcji (`php-mysql`).

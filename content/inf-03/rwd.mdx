Responsywność (RWD, Responsive Web Design) to projektowanie stron, które dobrze wyglądają na każdym ekranie — od 320px telefonu do 4K monitora. W tej lekcji poznasz techniki, dzięki którym Twoja strona będzie elastyczna.

## Filozofia: mobile-first

**Mobile-first** = zaczynaj projektowanie od najmniejszego ekranu, potem rozszerzaj. Odwrotnie niż "desktop z breakpoints w dół".

Dlaczego mobile-first:

- Większość ruchu w sieci to mobile (Google indeksuje mobile).
- Łatwiej upraszczać niż komplikować.
- CSS ładuje się szybciej (mniej reguł na start).

```css
/* Domyślne style — dla mobile */
.karta { padding: 12px; }

/* Rozszerzenie dla większych ekranów */
@media (min-width: 768px) {
    .karta { padding: 24px; }
}
```

NIE tak:

```css
/* Style desktopowe, a potem "cofaj" na mobile */
.karta { padding: 24px; }
@media (max-width: 768px) {
    .karta { padding: 12px; }
}
```

## Viewport meta — OBOWIĄZKOWE

Bez tego tagu strona na telefonie renderuje się jak na desktopie i user widzi miniaturę:

```html
<meta name="viewport" content="width=device-width, initial-scale=1">
```

- `width=device-width` — strona dopasowana do szerokości urządzenia.
- `initial-scale=1` — początkowe powiększenie 1 (bez zoomowania).

<Callout type="exam">
Na egzaminie INF.03 `<meta name="viewport" content="width=device-width, initial-scale=1">` to punkt obowiązkowy. Brak = strona się nie skaluje na telefonie = utrata punktów.
</Callout>

## Media queries — kiedy zmienić układ

```css
/* Domyślnie: mobile (pojedyncza kolumna) */
.kontener {
    display: grid;
    grid-template-columns: 1fr;
    gap: 16px;
}

/* Tablet (>= 768px) */
@media (min-width: 768px) {
    .kontener {
        grid-template-columns: 1fr 1fr;
    }
}

/* Desktop (>= 1024px) */
@media (min-width: 1024px) {
    .kontener {
        grid-template-columns: 1fr 1fr 1fr;
    }
}

/* Duży monitor (>= 1440px) */
@media (min-width: 1440px) {
    .kontener { max-width: 1400px; margin: 0 auto; }
}
```

Popularne breakpointy (orientacyjnie):

- `320px` — małe telefony
- `480px` — telefony
- `768px` — tablety (portrait)
- `1024px` — tablety (landscape), małe laptopy
- `1280px` — desktop
- `1440px` — duży desktop
- `1920px` — Full HD

Nie trzymaj się ich sztywno — dodawaj breakpoint gdy layout **naprawdę się psuje**.

## Jednostki elastyczne

Zamiast sztywnych `px`, używaj jednostek dopasowanych do ekranu:

```css
/* Sekcja hero wypełniająca ekran */
.hero {
    min-height: 100vh;        /* pełna wysokość okna */
    padding: 5vw;             /* odstęp 5% szerokości */
}

h1 {
    font-size: clamp(1.5rem, 4vw, 3rem);  /* min, preferowane, max */
}

.karta {
    width: 90%;               /* prawie cała szerokość na mobile */
    max-width: 400px;         /* nie szersza niż 400 na desktop */
}
```

`clamp(min, preferred, max)` — płynna typografia, idealna do nagłówków.

## Obrazy elastyczne

Trzy techniki, żeby obrazki nie wychodziły poza ekran:

```css
img {
    max-width: 100%;       /* nie szerszy niż kontener */
    height: auto;          /* zachowaj proporcje */
}
```

```html
<!-- Różne rozmiary dla różnych ekranów -->
<img src="hero-small.jpg"
     srcset="hero-small.jpg 480w,
             hero-medium.jpg 768w,
             hero-large.jpg 1440w"
     sizes="(max-width: 768px) 100vw, 50vw"
     alt="...">
```

```html
<!-- Sztuczka aspect-ratio bez rozmiaru obrazu -->
<img src="hero.jpg" style="aspect-ratio: 16/9; width: 100%; object-fit: cover;">
```

`object-fit: cover` przycina obrazek zachowując proporcje (jak tło `background-size: cover`).

## Ukrywanie/pokazywanie elementów

```css
.menu-mobile { display: block; }
.menu-desktop { display: none; }

@media (min-width: 768px) {
    .menu-mobile { display: none; }
    .menu-desktop { display: flex; }
}
```

<Callout type="warning" title="Nie ukrywaj treści">
Lepiej responsywnie zmienić układ niż ukrywać ważną treść. Wyszukiwarki i czytniki ekranu NIE lubią `display: none` na kluczowych rzeczach.
</Callout>

## Testowanie — DevTools

Chrome DevTools ma tryb responsywny (Ctrl+Shift+M / Cmd+Shift+M):

- Lista popularnych urządzeń (iPhone, iPad, Pixel).
- Dowolna szerokość — wpisz ręcznie.
- Tryb portrait/landscape.
- Throttling sieci — testuj na 3G.

Zawsze testuj:

1. Najmniejszy wspierany ekran (320px).
2. Najpopularniejszy telefon (375–414px).
3. Tablet portrait (768px).
4. Desktop (1280–1920px).

## Podsumowanie

`<meta name="viewport" content="width=device-width, initial-scale=1">` — obowiązkowe. Mobile-first: pisz style dla małych ekranów, rozszerzaj `@media (min-width: ...)`. Breakpointy: 768 (tablet), 1024 (laptop), 1280 (desktop), 1440 (duży ekran). Elastyczne jednostki: `clamp()`, `vw`, `%`, `min/max-width`. Obrazy: `max-width: 100%; height: auto` + `srcset`/`sizes`. Testuj w DevTools (Ctrl+Shift+M).

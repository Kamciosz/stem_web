Flexbox świetnie radzi sobie z jednym wymiarem — rzędem albo kolumną. Ale gdy chcesz zbudować layout całej strony: header na górze, sidebar po lewej, treść w środku, footer na dole — i żeby wszystko było idealnie wyrównane w wierszach **i** kolumnach jednocześnie — Flexbox zaczyna walczyć. Tu wkracza <Term word="grid">CSS Grid</Term>: system zaprojektowany do układów dwuwymiarowych. W tej lekcji zbudujesz prawdziwy layout strony.

## Flexbox vs Grid — kiedy czego używać

To pytanie pada na egzaminie i w pracy. Zasada jest prosta:

- **<Term word="flexbox">Flexbox</Term>** = jeden wymiar. Elementy w rzędzie LUB kolumnie. Pasek nawigacji, lista przycisków, centrowanie.
- **<Term word="grid">Grid</Term>** = dwa wymiary. Siatka wiersz × kolumna. Layout strony, galeria, dashboard, złożony formularz.

Nie ma "lepszego" — oba są dobre do swoich zadań. W praktyce łączysz je: Grid na makro (layout całej strony), Flexbox wewnątrz (np. pasek nawigacji w nagłówku).

## Włączenie Grid

```css
.kontener {
    display: grid;
}
```

Samo `display: grid` daje jedną kolumnę i tyle wierszy ile dzieci — mało użyteczne. Siłę Grida odblokowujesz dopiero definiując siatkę.

## Definiowanie kolumn i wierszy

```css
.kontener {
    display: grid;
    grid-template-columns: 200px 1fr 200px;  /* 3 kolumny */
    grid-template-rows: auto 1fr auto;       /* 3 wiersze */
    gap: 16px;                               /* odstępy */
}
```

Tu definiujesz siatkę: trzy kolumny (lewa 200px, środkowa elastyczna, prawa 200px) i trzy wiersze.

### Jednostki Grida

- **`px`**, **`%`** — sztywne lub procentowe.
- **`fr`** — "fraction", kawałek wolnej przestrzeni. `1fr 1fr 1fr` = trzy równe kolumny. `1fr 2fr` = druga dwa razy szersza.
- **`auto`** — dopasuj do zawartości.
- **`minmax(min, max)`** — zakres. `minmax(200px, 1fr)` = minimum 200px, rośnie do wypełnienia.
- **`repeat(ile, wzorzec)`** — powtórzenie. `repeat(3, 1fr)` = `1fr 1fr 1fr`.
- **`auto-fit` / `auto-fill`** — tyle kolumn ile się zmieści.

`fr` to serce Grida. Zamiast liczyć procenty, mówisz "podziel wolne miejsce w tych proporcjach".

```css
/* Responsywna galeria BEZ media queries */
.galeria {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
    gap: 16px;
}
```

To połączenie `auto-fit` + `minmax` daje **tyle kolumn, ile się zmieści**, każda minimum 250px szerokości. Okno szersze — więcej kolumn. Węższe — mniej. Bez ani jednej <Term word="media query">media query</Term>. Naucz się tego wzorca, jest niezwykle praktyczny.

## Named areas — siatka z nazwami

Zamiast operować numerami linii, możesz nazwać obszary siatki słowami:

```css
.layout {
    display: grid;
    grid-template-columns: 200px 1fr;
    grid-template-rows: auto 1fr auto;
    grid-template-areas:
        "header  header"
        "sidebar main"
        "footer  footer";
    gap: 16px;
    min-height: 100vh;
}

header { grid-area: header; }
aside  { grid-area: sidebar; }
main   { grid-area: main; }
footer { grid-area: footer; }
```

Spójrz na `grid-template-areas` — widzisz układ strony wprost w kodzie. Header zajmuje całą górę, sidebar i main są obok siebie, footer na dole przez całą szerokość. Gdy chcesz zmienić układ — przesuwasz słowa. Czytelne i łatwe w utrzymaniu.

## Umieszczanie dzieci w siatce

Trzy sposoby:

### 1. Automatyczny

Dzieci wypełniają wolne komórki po kolei. Nic nie robisz.

### 2. Numer linii

```css
.dziecko {
    grid-column: 1 / 3;  /* od linii 1 do 3 = 2 kolumny */
    grid-row: 2 / 4;     /* od linii 2 do 4 = 2 wiersze */
}
```

Linie numerowane są od 1. Skrót `grid-column: 1 / -1` znaczy "od początku do końca" — czyli cała szerokość siatki.

### 3. Named area

```css
.dziecko {
    grid-area: header;  /* wpada w obszar nazwany "header" */
}
```

## gap — odstępy

```css
.kontener {
    display: grid;
    gap: 16px;          /* wszędzie */
    column-gap: 24px;   /* tylko między kolumnami */
    row-gap: 12px;      /* tylko między wierszami */
    gap: 12px 24px;     /* wiersze | kolumny */
}
```

`gap` w Grid działa tak samo jak we Flexbox — żadnych hacków z marginesami na ostatnim elemencie.

## Wyrównanie zawartości w komórkach

```css
.kontener {
    display: grid;
    justify-items: stretch;  /* domyślne: rozciągnij do szerokości komórki */
    justify-items: center;   /* wyśrodkuj poziomo */
    align-items: center;     /* wyśrodkuj pionowo */
}

/* Dla konkretnego dziecka: */
.dziecko {
    justify-self: center;
    align-self: center;
}
```

## Przykład — layout strony

To najczęstsze zastosowanie Grida na egzaminie:

```html
<div class="layout">
    <header class="header">
        <img src="logo.png" alt="Logo">
        <nav>...</nav>
    </header>
    <aside class="sidebar">
        <h3>Moduły</h3>
        <ul>...</ul>
    </aside>
    <main class="main">
        <article>...</article>
    </main>
    <footer class="footer">© 2026 Koło STEM</footer>
</div>
```

```css
.layout {
    display: grid;
    grid-template-columns: 240px 1fr;
    grid-template-rows: auto 1fr auto;
    grid-template-areas:
        "header  header"
        "sidebar main"
        "footer  footer";
    gap: 16px;
    min-height: 100vh;
    padding: 16px;
}

.header  { grid-area: header; display: flex; align-items: center; gap: 16px; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

/* Na telefonie — wszystko w jednej kolumnie */
@media (max-width: 768px) {
    .layout {
        grid-template-columns: 1fr;
        grid-template-areas:
            "header"
            "main"
            "sidebar"
            "footer";
    }
}
```

Zobacz, jak elegancko działa responsywność: na desktopie dwie kolumny, na telefonie jedna — zmieniasz tylko `grid-template-areas`. Header w nagłówku używa Flexboxa. Grid i Flexbox współpracują.

<Callout type="exam">
Na egzaminie INF.03 zadania z CSS często wymagają Grida do layoutu strony. Zapamiętaj wzorzec: `grid-template-areas` na kontenerze + `grid-area` na dzieciach + responsywność przez zmianę `grid-template-areas` w media query. To trzy elementy, które musisz umieć napisać z pamięci.
</Callout>

## Przykład — dashboard z kartami różnej wielkości

```html
<div class="dashboard">
    <div class="karta duza">1</div>
    <div class="karta">2</div>
    <div class="karta">3</div>
    <div class="karta">4</div>
    <div class="karta szeroka">5</div>
</div>
```

```css
.dashboard {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    grid-auto-rows: 150px;
    gap: 16px;
}

.duza    { grid-column: span 2; grid-row: span 2; }  /* 2×2 */
.szeroka { grid-column: span 3; }                     /* cała szerokość */
```

`grid-column: span 2` = "zajmij 2 kolumny". `grid-row: span 2` = "zajmij 2 wiersze". Tak budujesz nieregularne układy, gdzie jedna karta jest większa od innych.

## Podsumowanie

`display: grid` włącza <Term word="grid">Grid</Term>. `grid-template-columns` / `grid-template-rows` definiują siatkę. Jednostka `fr` dzieli wolną przestrzeń, `minmax()` daje zakres, `repeat(auto-fit, minmax(250px, 1fr))` robi responsywną galerię bez media queries. `grid-template-areas` + `grid-area` na dzieciach = czytelny layout. `gap` daje odstępy. Grid to dwa wymiary (wiersze + kolumny), Flexbox jeden (rząd LUB kolumna) — łącz je świadomie.

## Jak wygląda egzamin — struktura

Egzamin INF.03 ma dwie części, zdawane **niezależnie** (jedna nie wpływa na drugą):

1. **Część pisemna** — 40 pytań zamkniętych z HTML, CSS, JS, PHP, SQL, baz danych, bezpieczeństwa. Czas: 60 minut. Za każdą poprawną odpowiedź 1 punkt.
2. **Część praktyczna** — jedno duże zadanie: strona + baza + <Term word="backend">backend</Term>. Czas: 150 minut. Punkty za konkretne elementy.

Możesz nie zdać pisemnej i zdać praktyczną — i odwrotnie. Końcowy wynik to suma.

## Co dostajesz w części praktycznej

Typowe zadanie to prosty system webowy — rezerwacje, sklep, ogłoszenia, rejestr klientów. Zawsze dostajesz:

- **Wymagania** — co system ma robić.
- **Makiety** — jak ma wyglądać (kolory, layout, sekcje).
- **Schemat bazy** — tabele, kolumny, relacje.
- **Kryteria oceniania** — dokładna punktacja za każdy element.

Egzaminator porównuje Twój wynik z wzorcem punkt po punkcie. Nie ma "za aestetykę" — jest "za poprawne połączenie z bazą", "za prepared statements", "za walidację".

## Szablony punktowane na każdym egzaminie

Niezależnie od tematu zadania, te rzeczy są punktowane ZAWSZE:

### 1. Połączenie z bazą (3-5 pkt)

```php
<?php
// db.php — jedyny plik z konfiguracją
$host = "localhost";
$dbname = "egzamin";
$user = "root";
$pass = "";

try {
    $pdo = new PDO(
        "mysql:host=$host;dbname=$dbname;charset=utf8mb4",
        $user, $pass,
        [
            PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
            PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
            PDO::ATTR_EMULATE_PREPARES => false
        ]
    );
} catch (PDOException $e) {
    die("Błąd: " . htmlspecialchars($e->getMessage()));
}
```

W każdym pliku: `require_once "db.php";`. Gotowe, 5 punktów.

### 2. Prepared statements WSZĘDZIE (3-5 pkt)

```php
// Zamiast:
$pdo->query("SELECT * FROM users WHERE email = '$email'");  // ❌ SQL injection!

// Użyj:
$stmt = $pdo->prepare("SELECT * FROM users WHERE email = :email");
$stmt->execute([":email" => $_GET["email"]]);
```

<Term word="SQL injection">SQL injection</Term> = natychmiastowa utrata punktów. Egzaminator sprawdza to jako **osobne kryterium**.

### 3. htmlspecialchars() przy KAŻDYM echo (2-3 pkt)

```php
<!-- NIGDY nie wyświetlaj danych bezpośrednio -->
<?= htmlspecialchars($user["name"]) ?>
<?= htmlspecialchars($_POST["email"] ?? "") ?>
```

<Term word="XSS">XSS</Term> = punkty ujemne. To dwie sekundy, żeby zawinąć w htmlspecialchars.

### 4. Walidacja HTML5 (2-3 pkt za darmo)

```html
<form method="post">
    <input name="email" type="email" required>
    <input name="age" type="number" min="13" max="120" required>
    <input name="phone" type="tel" pattern="[0-9]{9}" required>
    <button>Wyślij</button>
</form>
```

### 5. Walidacja PHP po stronie serwera (3-5 pkt)

```php
$bledy = [];
if (empty($_POST["data"])) $bledy[] = "Data jest wymagana";
if (!is_numeric($_POST["sala_id"] ?? "")) $bledy[] = "Nieprawidłowa sala";
if (empty($_POST["imie"])) $bledy[] = "Imię jest wymagane";
```

Egzaminator sprawdza obie strony. Sama walidacja HTML5 = za mało.

### 6. Responsywność minimum (2-3 pkt)

```css
.karta { width: 100%; }
@media (min-width: 768px) {
    .karta { width: 50%; }
}
```

## 5 błędów, które powtarzają się na każdym egzaminie

### 1. Brak WHERE w UPDATE/DELETE

```php
// ❌ Wszyscy userzy = admin
$pdo->exec("UPDATE users SET role = 'admin'");

// ✅ Tylko ten jeden
$stmt = $pdo->prepare("UPDATE users SET role = 'admin' WHERE id = :id");
$stmt->execute([":id" => $userId]);
```

### 2. Dane z formularza prosto do SQL

```php
// ❌ SQL injection
$pdo->query("SELECT * FROM users WHERE email = '$email'");

// ✅ Prepared statement
$stmt = $pdo->prepare("SELECT * FROM users WHERE email = :email");
$stmt->execute([":email" => $_POST["email"]]);
```

### 3. Brak htmlspecialchars()

```php
// ❌ XSS
echo "Witaj, $imie";

// ✅ Bezpiecznie
echo "Witaj, " . htmlspecialchars($imie);
```

### 4. session_start() po HTML

```php
<!-- ❌ Błąd headers already sent -->
<html>
<?php session_start(); ?>

<!-- ✅ session_start na samej górze pliku -->
<?php session_start(); ?>
<html>
```

### 5. Brak redirect po POST

```php
// ❌ F5 = ponowne wysłanie formularza
$pdo->prepare("INSERT INTO rezerwacje ...")->execute([...]);
echo "Zapisano!";

// ✅ Post/Redirect/Get
$pdo->prepare("INSERT INTO rezerwacje ...")->execute([...]);
header("Location: index.php");
exit;
```

<Callout type="exam">
Post/Redirect/Get (PRG) = po INSERT/UPDATE/DELETE zawsze rób `header("Location: ...")` + `exit`. Bez tego F5 powtarza operację.
</Callout>

## Ostatnie 15 minut — NIE pisz nowego kodu

Gdy czas się kończy:

1. Przejdź CAŁY arkusz — czy spełniłeś każdy wymóg?
2. Sprawdź każdy formularz — czy działa, czy waliduje.
3. Sprawdź każde echo — czy htmlspecialchars.
4. Sprawdź każde zapytanie SQL — czy prepared statement.
5. Usuń var_dump / console.log / die("test").
6. Upewnij się, że db.php jest require_once, nie require.

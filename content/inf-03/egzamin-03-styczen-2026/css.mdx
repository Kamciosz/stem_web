<Callout type="exam" title="Zasada oceniania CSS">
Najpierw selektory z polecenia, potem wygląd. Jeżeli polecenie wymaga selektora znacznika, klasy albo konkretnego wymiaru, nie zastępuj tego 'lepszym' rozwiązaniem.
</Callout>

## Kolejność pracy nad stylem

1. Ustaw globalne fonty i kolory tła.
2. Zbuduj układ głównych sekcji: header, nav, main, aside/section, footer.
3. Dopiero potem dopracuj obrazy, tabele, linki, formularze i paragrafy.
4. Na końcu porównaj z makietą: wysokości, szerokości, wyrównania, obramowania i overflow.

<Callout type="warning" title="Za co bym obciął">
Za `ładniej niż w makiecie`, za pominięcie `overflow`, za zły font, za brak wymaganej ramki, za użycie klasy tam, gdzie polecenie wymaga selektora znacznika, i za obraz w złym rozmiarze.
</Callout>

## CSS — pełne rozwiązanie (styl.css)

```css
* {
    font-family: Garamond, serif;
}

header, footer {
    background-color: Navy;
    color: white;
    text-align: center;
    padding: 5px;
    letter-spacing: 3px;
}

main {
    display: flex;
}

nav, section, aside {
    height: 500px;
}

nav, aside {
    background-color: DodgerBlue;
    width: 35%;
}

section {
    background-color: SkyBlue;
    width: 30%;
    text-align: center;
}

input, button {
    margin: 10px;
}

h3 {
    text-align: center;
}

h3::first-letter {
    font-size: 200%;
}

p {
    font-size: 150%;
}

.miniatura {
    box-shadow: 5px 5px 10px DimGray;
    border-radius: 10px;
    margin: 5px;
    cursor: pointer;
}
```

<Callout type="exam">
`h3::first-letter` — pseudo-element, który powiększa pierwszą literę nagłówka do 200%. To rzadko spotykane na egzaminie, ale polecenie mówi wprost. Zapamiętaj składnię z podwójnym dwukropkiem.
</Callout>

## Mini-rubryka CSS

Weryfikator nie czyta twoich intencji. Patrzy na efekt i wymagania:

- selektor jest taki, jak w poleceniu;
- wymiar nie jest przypadkowy, tylko zgodny z arkuszem;
- kolor, font i obramowanie są wpisane jawnie;
- obraz nie rozciąga się bez zachowania proporcji;
- układ działa po odświeżeniu, nie tylko po ręcznej zmianie w DevTools.

Jeżeli któryś punkt jest „prawie”, to jest do poprawy. Na INF.03 „prawie jak makieta” oznacza utratę punktów za szczegół, który był wprost opisany.

## Test zgodności z makietą

Nie oceniaj CSS po tym, czy strona „wygląda dobrze”. Oceniaj po zgodności z arkuszem:

1. Najpierw porównaj duże bloki: header, menu, main, sekcje boczne, footer.
2. Potem porównaj typografię: font, rozmiar, pogrubienie, kursywa, wyrównanie.
3. Następnie obrazy: rozmiar, obramowanie, zaokrąglenie, margines, pozycja.
4. Na końcu formularze i tabele: czy są w dobrym miejscu i czy nie rozpychają układu.

<Callout type="warning" title="To nie jest prawie dobrze">
Jeżeli polecenie mówi `height: 150px` i `overflow: hidden`, a ty dajesz automatyczną wysokość, to nie jest „responsywność”. To brak wymaganego stylu. Tak samo z kolorami, selektorami i rozmiarami obrazów.
</Callout>

## Co zostawić na koniec

Na końcu wolno poprawiać tylko rzeczy punktowane: brakujący kolor, zły margines, zły rozmiar obrazka, niedziałające ustawienie `display`, niezgodny selector. Nie zmieniaj całej struktury strony w ostatnich minutach, bo zepsujesz działające wymagania.

## Kontrola CSS przed przejściem dalej

- strona nie ma poziomego scrolla;
- główne bloki mają wysokości i szerokości zgodne z poleceniem;
- grafiki są w dobrych folderach i mają poprawne `src`;
- formularze, tabele i paragrafy dziedziczą style zgodnie z selektorami;
- po odświeżeniu strony wygląd nadal jest taki sam, nie tylko po ręcznej zmianie w DevTools.

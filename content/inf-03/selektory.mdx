Wyobraź sobie, że masz sto elementów na stronie i chcesz zmienić kolor tylko tych, które są linkami w nawigacji. Możesz dodać `style=""` do każdego z osobna — albo napisać jeden selektor CSS, który trafi dokładnie w te elementy. To właśnie jest sedno CSS: zamiast stylować każdy element ręcznie, opisujesz *które* elementy chcesz zmienić, a przeglądarka robi resztę.

## Jak wygląda reguła CSS

Każda reguła to selektor + blok deklaracji:

```css
selektor {
    właściwość: wartość;
    inna-właściwość: wartość;
}
```

Przykład — wszystkie nagłówki `<h1>` będą duże i fioletowe:

```css
h1 {
    color: #7c3aed;
    font-size: 48px;
    font-weight: 700;
}
```

Przeglądarka czyta Twój CSS, znajduje pasujące elementy w <Term word="DOM">drzewie DOM</Term> i stosuje style. Proste — dopóki reguły nie zaczną się gryźć. O tym za chwilę.

## Trzy sposoby dołączenia CSS do HTML

### 1. Zewnętrzny arkusz — zawsze tak

```html
<head>
    <link rel="stylesheet" href="style.css">
</head>
```

CSS żyje w osobnym pliku. Jeden plik styluje wiele stron, przeglądarka go cache'uje — ładuje raz, używa wszędzie.

### 2. Blok `<style>` — wyjątkowo

```html
<head>
    <style>
        h1 { color: red; }
    </style>
</head>
```

Dobre dla małych, jednorazowych eksperymentów. Na produkcji — zewnętrzny arkusz.

### 3. Inline `style=""` — prawie nigdy

```html
<h1 style="color: red;">Tytuł</h1>
```

Miesza HTML z CSS, trudno nadpisać, niespójne. Jedyny uzasadniony przypadek: dynamiczne style z JavaScriptu.

<Callout type="tip" title="Zasada kciuka">
Domyślnie: zewnętrzny arkusz. Wyjątkowo: `<style>` dla jednej strony. Inline tylko gdy JavaScript musi zmienić styl w locie.
</Callout>

## Selektory podstawowe — cztery typy

```css
/* Znacznik — trafia we wszystkie <p> */
p { color: #ccc; }

/* Klasa — trafia w elementy z class="karta" */
.karta { background: #222; }

/* ID — trafia w element z id="menu" */
#menu { position: fixed; top: 0; }

/* Uniwersalny — trafia we wszystko */
* { box-sizing: border-box; }
```

Klasy możesz używać wielokrotnie na stronie. ID powinno być unikalne — jeden element, jeden ID. W praktyce do stylowania używaj klas, ID zostaw dla JavaScriptu i kotwic.

## Selektory atrybutów

Wybierają elementy na podstawie <Term word="atrybut">atrybutu</Term> lub jego wartości:

```css
input[type="email"] { border: 2px solid blue; }
input[required]     { background: #fffacd; }

a[href^="https://"] { color: green; }    /* zaczyna się od */
a[href$=".pdf"]     { color: red; }      /* kończy się na */
a[href*="kontakt"]  { font-weight: bold; } /* zawiera */
```

Przydatne gdy nie możesz dodać klasy do elementu (np. generowany HTML).

## Pseudoklasy — stan elementu

Pseudoklasa opisuje stan, w jakim element się znajduje. Piszesz `:nazwa` po selektorze:

```css
a:hover   { color: yellow; }            /* kursor nad linkiem */
a:active  { color: red; }               /* wciśnięty */
a:visited { color: purple; }            /* odwiedzony */

input:focus    { border: 2px solid blue; } /* fokus (Tab lub klik) */
button:disabled { opacity: 0.5; }          /* wyłączony przycisk */

li:first-child  { font-weight: bold; }  /* pierwszy w rodzeństwie */
li:last-child   { border: none; }       /* ostatni */
li:nth-child(3) { color: gold; }        /* trzeci */
li:nth-child(odd)  { background: #111; } /* nieparzyste */
li:nth-child(even) { background: #222; } /* parzyste */
```

`:nth-child(odd/even)` to klasyk przy stylowaniu tabel — naprzemienne kolory wierszy bez JavaScriptu.

## Kombinatory — relacje między elementami

Kombinatory pozwalają wybrać element na podstawie jego pozycji względem innych:

```css
/* Potomek — a gdziekolwiek wewnątrz nav */
nav a { color: white; }

/* Dziecko — li będące bezpośrednim dzieckiem ul */
ul > li { list-style: square; }

/* Brat bezpośredni — p tuż po h1 */
h1 + p { font-size: 1.2em; }

/* Brat ogólny — wszystkie p po h1 (nie tylko pierwsze) */
h1 ~ p { color: #888; }
```

Najczęściej używasz `nav a` (potomek) i `ul > li` (dziecko). Reszta przydaje się rzadziej, ale warto znać.

### Grupowanie — wiele selektorów naraz

```css
h1, h2, h3 { font-family: 'Space Grotesk', sans-serif; }
.btn, .button, a.btn { padding: 10px 20px; }
```

Przecinek = "i to, i tamto". Zamiast pisać tę samą deklarację trzy razy — piszesz raz.

## Kaskada — co wygrywa gdy reguły się gryzą

CSS to *Cascading* Style Sheets — "kaskadowe". Gdy kilka reguł dotyczy tego samego elementu, przeglądarka musi zdecydować, która wygrywa. Decydują trzy rzeczy po kolei.

### 1. Specyficzność — waga selektora

To najważniejszy mechanizm. Każdy selektor ma wagę (a, b, c, d):

- **a** = inline `style=""` — najwyższa waga
- **b** = liczba ID (`#menu`)
- **c** = liczba klas, pseudoklas, atrybutów (`.karta`, `:hover`, `[type]`)
- **d** = liczba znaczników i pseudoelementów (`p`, `::before`)

```css
p              { color: red; }    /* (0,0,0,1) */
.karta         { color: green; }  /* (0,0,1,0) */
#menu          { color: blue; }   /* (0,1,0,0) */
.karta p:hover { color: yellow; } /* (0,0,2,1) */
```

`#menu` (0,1,0,0) wygrywa z `.karta` (0,0,1,0), bo ID > klasa.

<Callout type="warning" title="Inline vs ID">
Inline `style=""` ma wagę (1,0,0,0) — wygrywa z każdym ID bez `!important`. Dlatego inline jest trudny do nadpisania w arkuszu stylów. Unikaj go w HTML.
</Callout>

### 2. Kolejność w kodzie

Gdy specyficzność jest równa, wygrywa reguła zdefiniowana **później**:

```css
.btn { background: red; }
.btn { background: blue; }  /* ← wygrywa */
```

Dlatego style ogólne piszesz na górze pliku, a nadpisania na dole.

### 3. `!important` — ostateczność

```css
.btn { background: red !important; }
```

Przebija WSZYSTKO. Używaj tylko gdy nadpisujesz style zewnętrznego <Term word="framework">frameworka</Term> (Bootstrap, Tailwind). W normalnym kodzie `!important` to sygnał, że architektura CSS jest zepsuta — zamiast naprawiać problem, zaklejasz go taśmą.

## Dziedziczenie — co dziecko dostaje od rodzica

Niektóre właściwości **dziedziczą się** automatycznie: `color`, `font-family`, `font-size`, `line-height`. Inne NIE: `background`, `border`, `padding`, `margin`.

```css
body {
    color: #ccc;          /* dziedziczą <h1>, <p>, <a> */
    font-family: sans-serif; /* też dziedziczą */
    background: #000;     /* NIE dziedziczy */
}
```

Możesz wymusić lub wyzerować dziedziczenie:

```css
.btn { color: inherit; }      /* wymuś dziedziczenie koloru */
.btn { background: initial; } /* wyzeruj do wartości domyślnej */
```

## Debugowanie w DevTools

Gdy "coś nie działa" — nie zgaduj, sprawdź:

1. Otwórz DevTools (F12 lub Cmd+Opt+I).
2. Zakładka **Elements** — kliknij element, po prawej zobaczysz jego CSS.
3. Reguły **przekreślone** = nadpisane przez inną regułę o wyższej specyficzności.
4. Zakładka **Computed** — finalne wartości po zastosowaniu kaskady.

<Callout type="tip" title="Triki DevTools">
Ikona `:hov` w panelu Styles wymusza stany `:hover` / `:focus` bez ruszania myszą. Przydatne gdy chcesz zobaczyć styl przycisku po najechaniu.
</Callout>

<Callout type="exam">
Na egzaminie INF.03 pytają o specyficzność: inline > ID > klasa > znacznik. Pamiętaj też, że `!important` przebija wszystko, a przy równej specyficzności wygrywa późniejsza reguła. Umiej też wskazać, który selektor trafi w dany element HTML.
</Callout>

## Podsumowanie

Selektor wybiera elementy (znacznik, klasa, ID, atrybut, pseudoklasa, kombinator). Gdy dwie reguły mówią to samo, decyduje specyficzność: inline > ID > klasa > znacznik. Przy równej specyficzności wygrywa późniejsza. Dziedziczenie działa dla koloru i fontu — nie dla tła, marginesu, paddingu. `!important` to ostateczność, nie styl pracy. Gdy coś nie działa — DevTools, zakładka Elements, szukaj przekreślonych reguł.

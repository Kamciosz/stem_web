CSS to język, który mówi przeglądarce **jak ma wyglądać** strona. Reguły CSS wybierają elementy HTML (selektory) i nadają im style. Gdy dwie reguły mówią to samo, decyduje kaskada i specyficzność. W tej lekcji opanujesz oba mechanizmy.

## Jak działa reguła CSS

```css
selektor {
    właściwość: wartość;
    inna-właściwość: wartość;
}
```

Przykład — wszystkie nagłówki `<h1>` będą duże i fioletowe:

```css
h1 {
    color: #7c3aed;
    font-size: 48px;
    font-weight: 700;
}
```

Przeglądarka czyta Twój CSS, znajduje pasujące elementy, stosuje style. Proste — dopóki reguły nie zaczną się gryźć.

## Sposoby dodawania CSS do HTML

### 1. Zewnętrzny arkusz (najlepsza praktyka)

```html
<head>
    <link rel="stylesheet" href="style.css">
</head>
```

CSS żyje w osobnym pliku. Można go reużyć na wielu stronach, łatwo cache'ować.

### 2. Wewnątrz `<style>` (dla pojedynczej strony)

```html
<head>
    <style>
        h1 { color: red; }
    </style>
</head>
```

Dobre dla małych, jednorazowych stylów.

### 3. Inline `style=""` (ostateczność)

```html
<h1 style="color: red;">Tytuł</h1>
```

**Unikaj** — miesza HTML z CSS, trudno nadpisać, niespójne.

<Callout type="tip" title="Zasada thumb-of-rule">
Domyślnie: zewnętrzny arkusz. Wyjątkowo: `<style>` dla jednej strony. Inline tylko dla dynamicznych stylów z JavaScriptu.
</Callout>

## Selektory — jak wybrać elementy

### Selektory podstawowe

```css
/* Znacznik */
p { color: #ccc; }

/* Klasa (atrybut class="...") */
.karta { background: #222; }

/* Id (atrybut id="...") */
#menu { position: fixed; top: 0; }

/* Uniwersalny (wszystko) */
* { box-sizing: border-box; }
```

### Selektory atrybutów

Wybierają elementy na podstawie atrybutu lub jego wartości:

```css
input[type="email"] { border: 2px solid blue; }
input[required] { background: #fffacd; }
a[href^="https://"] { color: green; }     /* zaczyna się od */
a[href$=".pdf"] { color: red; }            /* kończy się na */
a[href*="kontakt"] { font-weight: bold; }  /* zawiera */
```

### Pseudoklasy — stan elementu

Najczęstsze:

```css
a:hover { color: yellow; }            /* kursor nad */
a:active { color: red; }              /* wciśnięty */
a:visited { color: purple; }          /* odwiedzony */
input:focus { border: 2px solid blue; } /* fokus (klawiatura/Tab) */
button:disabled { opacity: 0.5; }     /* wyłączony */
li:first-child { font-weight: bold; } /* pierwszy w rodzeństwie */
li:last-child { }                     /* ostatni */
li:nth-child(3) { }                   /* trzeci */
li:nth-child(odd) { }                 /* nieparzyste */
li:nth-child(even) { }                /* parzyste */
```

### Kombinatory — relacje między elementami

```css
/* Potomek (gdziekolwiek w dół) */
nav a { color: white; }

/* Dziecko (bezpośredni potomek) */
ul > li { list-style: square; }

/* Brat (bezpośrednio po) */
h1 + p { font-size: 1.2em; }

/* Brat (gdziekolwiek po) */
h1 ~ p { color: #888; }
```

### Grupowanie — wiele selektorów naraz

```css
h1, h2, h3 { font-family: 'Space Grotesk', sans-serif; }
.btn, .button, a.btn { padding: 10px 20px; }
```

## Kaskada — co wygrywa

CSS to **Cascading Style Sheets** — "kaskadowe". Gdy kilka reguł dotyczy tego samego elementu, przeglądarka musi zdecydować, która wygrywa. Decydują trzy rzeczy (w kolejności):

### 1. Pochodzenie (origin)

- **Autor strony** — Twój CSS.
- **Użytkownik** — np. własna karta stylów.
- **Przeglądarka** — domyślne style (np. niebieskie linki).

Autor wygrywa z domyślnymi, ale `!important` użytkownika przebije autora.

### 2. Specyficzność (specificity) — NAJWAŻNIEJSZE

Specyficzność to "waga" selektora. Wyższa waga wygrywa. Liczona jest jako (a, b, c):

- **a** = inline style (`style="..."`) — liczy się za 1, 0, 0, 0
- **b** = liczba ID
- **c** = liczba klas, pseudoklas, atrybutów
- **d** = liczba znaczników, pseudoelementów

```css
/* specyficzność (0, 0, 0, 1) */
p { color: red; }

/* specyficzność (0, 0, 1, 0) */
.karta { color: green; }

/* specyficzność (0, 1, 0, 0) */
#menu { color: blue; }

/* specyficzność (0, 0, 2, 1) */
.karta p:hover { color: yellow; }
```

Wygrywa wyższa specyficzność. `#menu` (0,1,0,0) > `.karta` (0,0,1,0) > `p` (0,0,0,1).

<Callout type="warning" title="Inline vs ID">
Inline `style="..."` ma specyficzność (1,0,0,0) — wygrywa z ID bez `!important`. Dlatego inline jest trudny do nadpisania — unikaj go.
</Callout>

### 3. Kolejność w kodzie (source order)

Gdy specyficzność jest równa, wygrywa reguła **zdefiniowana później**:

```css
.btn { background: red; }
.btn { background: blue; }  /* ← wygrywa (definiowana później) */
```

### `!important` — ostateczność

```css
.btn { background: red !important; }
```

`!important` przebija WSZYSTKO. Używaj tylko gdy:
- nadpisujesz style frameworka (Bootstrap, Tailwind reset)
- tworzysz narzędzie deweloperskie

W normalnym kodzie `!important` to sygnał, że architektura CSS jest zepsuta.

## Dziedziczenie — co przejmuje dziecko

Niektóre właściwości **dziedziczą się** z rodzica: `color`, `font-family`, `font-size`, `line-height`. Inne NIE: `background`, `border`, `padding`, `margin`.

```css
body {
    color: #ccc;       /* dziedziczą <h1>, <p>, <a> */
    font-family: sans-serif;  /* też dziedziczą */
    background: #000;  /* NIE dziedziczy (background body = tło strony) */
}
```

Wymuszenie dziedziczenia:

```css
.btn { color: inherit; }   /* wymuś dziedziczenie koloru */
.btn { background: initial; } /* wyzeruj */
```

## Debugowanie — narzędzia przeglądarki

Gdy "coś nie działa":

1. **Otwórz DevTools** (F12 lub Cmd+Opt+I).
2. Zakładka **Elements** — kliknij element, zobaczysz jego CSS po prawej.
3. Reguły przekreślone = nadpisane przez inne.
4. Na górze sekcji **Computed** — finalne wartości.
5. Na górze sekcji **Styles** — kaskada z informacją, skąd każda wartość.

<Callout type="tip" title="Triki DevTools">
- Zaznacz element, kliknij jego `padding` w panelu Computed — zobaczysz wizualizację.
- W zakładce Styles możesz edytować wartości na żywo (nie zapisuje się).
- `Toggle element state` (ikona `:hov` w prawym górnym rogu Styles) wymusza `:hover` / `:focus`.
</Callout>

## Podsumowanie

Selektor wybiera elementy (znacznik, klasa, id, atrybut, pseudoklasa, kombinator). Właściwość + wartość = deklaracja. Gdy dwie reguły mówią to samo, decyduje specyficzność: inline > ID > klasa > znacznik. Przy równej specyficzności wygrywa późniejsza. Dziedziczenie działa dla koloru, fontu, line-height; nie działa dla tła, marginesu, paddingu. `!important` to ostateczność, nie styl pracy.

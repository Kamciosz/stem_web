```sql
CREATE TABLE kolory (
  id      INT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  nazwa   VARCHAR(20) NOT NULL,
  doplata DECIMAL(10,2) UNSIGNED DEFAULT NULL
);

CREATE TABLE pojazdy (
  id    INT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  marka VARCHAR(20) NOT NULL,
  model VARCHAR(20) NOT NULL,
  cena  DECIMAL(15,0) UNSIGNED NOT NULL,
  kolor INT UNSIGNED NOT NULL,
  FOREIGN KEY (kolor) REFERENCES kolory(id)
);
```

**Dane:** 8 kolorów (4 bazowe z dopłatą 0, 4 metalik z dopłatą 1000). 62 pojazdy, marki `BM`/`AM`, modele `alfa`/`beta`/`meta`.

## Zadanie 1 — Zapytania SQL

Egzaminator daje 4 zapytania. Oto one z rozwiązaniem:

### Zapytanie 1: 15 najdroższych pojazdów marki BM

```sql
SELECT marka, model, cena
FROM pojazdy
WHERE marka = 'BM'
ORDER BY cena DESC
LIMIT 15;
```

<Callout type="tip" title="Punkty za kolejność">
`ORDER BY` MUSI być przed `LIMIT`. Odwrotna kolejność = błąd składni = 0 punktów za to zapytanie.
</Callout>

### Zapytanie 2: średnia i maksymalna cena modelu "meta"

```sql
SELECT AVG(cena) AS 'Średnia cena', MAX(cena) AS 'Maksymalna cena'
FROM pojazdy
WHERE model = 'meta';
```

Aliasy (`AS`) muszą być **dokładnie** takie jak w poleceniu. Egzaminator porównuje wynik z wzorcem.

### Zapytanie 3: pojazdy "alfa" z danymi koloru (JOIN)

```sql
SELECT marka, model, cena, nazwa, doplata
FROM pojazdy
INNER JOIN kolory ON pojazdy.kolor = kolory.id
WHERE model = 'alfa';
```

<Callout type="exam">
<Term word="JOIN">JOIN</Term> na kluczu obcym to **najczęściej punktowane zapytanie** na egzaminie INF.03. Jeśli nie umiesz pisać JOIN-ów z pamięci — wróć do lekcji "Relacje i JOIN" i ćwicz do skutku.
</Callout>

### Zapytanie 4: 2 losowe pojazdy

```sql
SELECT marka, model, cena
FROM pojazdy
ORDER BY RAND()
LIMIT 2;
```

`ORDER BY RAND()` — sortuje losowo, `LIMIT 2` bierze dwa pierwsze. Proste, ale trzeba znać.

### Sekcja #lewa — tabela pojazdów "alfa" z ceną całkowitą

Tu łączysz `pojazdy` z `kolory` i obliczasz cenę w PHP:

```php
$query = "SELECT marka, model, cena, nazwa, doplata
          FROM pojazdy
          INNER JOIN kolory ON kolor = kolory.id
          WHERE model = 'alfa'";
$result = $conn->query($query);

echo "<table>";
echo "<tr><th>Marka</th><th>Model</th><th>Kolor</th><th>Cena całkowita</th></tr>";

while ($row = $result->fetch_assoc()) {
    $cena_calkowita = $row['cena'] + $row['doplata'];
    echo "<tr>";
    echo "<td>" . $row['marka'] . "</td>";
    echo "<td>" . $row['model'] . "</td>";
    echo "<td>" . $row['nazwa'] . "</td>";
    echo "<td>" . $cena_calkowita . "</td>";
    echo "</tr>";
}
echo "</table>";
```

**Dlaczego obliczenie w PHP a nie w SQL?** Bo polecenie mówi "oblicz cenę całkowitą w skrypcie". Egzaminator sprawdza, CZY robisz to w PHP. Gdybyś zrobił `SELECT cena + doplata AS cena_calkowita` w SQL — technicznie działa, ale tracisz punkt za "realizację w skrypcie".

## Wnioski z sesji styczeń 2026

Co nowego / nietypowego w porównaniu z poprzednimi latami:

1. **Obliczenie w PHP zamiast SQL** — wcześniej (2024-2025) cena finalna była liczona w zapytaniu SQL (`SELECT cena + doplata`). Teraz egzaminator wymaga, żebyś to zrobił w pętli PHP. Czytaj polecenie dosłownie.

2. **ORDER BY RAND()** — pojawiło się po raz pierwszy w sesji głównej (wcześniej było tylko w próbnych). Zapamiętaj składnię.

3. **rowspan w dynamicznej tabeli** — generujesz HTML z `rowspan` wewnątrz pętli `while`. To trudniejsze niż statyczny rowspan, bo musisz pilnować, który wiersz ma `rowspan` a który nie.

4. **Motyw kolorystyczny z nazwami CSS** — `DarkGreen`, `ForestGreen`, `LightGreen`. Egzaminator akceptuje zarówno nazwy jak i hex, ale nazwy są szybsze do wpisania i mniej podatne na literówki.

5. **overflow: auto na section** — gdy tabela jest dłuższa niż 550px, pojawia się scroll. Bez tego treść wylewa się poza sekcję i tracisz punkt za "zgodność z makietą".

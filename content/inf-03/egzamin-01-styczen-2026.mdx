<div className="exam-verdict warning">
  <div className="exam-verdict-label">punkt krytyczny</div>
  <div className="exam-copy">
    Cena całkowita ma być policzona w PHP: <code>pojazdy.cena + kolory.doplata</code>.
    Jak policzysz ją w SQL, strona zadziała, ale realizacja polecenia będzie zła.
  </div>

```sql
CREATE TABLE kolory (
  id      INT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  nazwa   VARCHAR(20) NOT NULL,
  doplata DECIMAL(10,2) UNSIGNED DEFAULT NULL
);

CREATE TABLE pojazdy (
  id    INT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  marka VARCHAR(20) NOT NULL,
  model VARCHAR(20) NOT NULL,
  cena  DECIMAL(15,0) UNSIGNED NOT NULL,
  kolor INT UNSIGNED NOT NULL,
  FOREIGN KEY (kolor) REFERENCES kolory(id)
);
```

## 5. SQL — cztery zapytania i zero filozofii

<div className="exam-solution-block">
  <div className="solution-head">
    <span>ZAPYTANIE 1</span>
    <h3>15 najdroższych pojazdów marki BM</h3>
  </div>

```sql
SELECT marka, model, cena
FROM pojazdy
WHERE marka = 'BM'
ORDER BY cena DESC
LIMIT 15;
```

  <div className="solution-check good">
    <strong>Dlaczego to przechodzi:</strong> filtrujesz markę, sortujesz malejąco po cenie i dopiero potem ograniczasz do 15.
  </div>
  <div className="solution-check bad">
    <strong>Za co bym obciął:</strong> <code>LIMIT</code> przed <code>ORDER BY</code>, brak <code>DESC</code>, literówka w <code>BM</code>.
  </div>
</div>

<div className="exam-solution-block">
  <div className="solution-head">
    <span>ZAPYTANIE 2</span>
    <h3>Średnia i maksymalna cena modelu meta</h3>
  </div>

```sql
SELECT AVG(cena) AS 'Średnia cena', MAX(cena) AS 'Maksymalna cena'
FROM pojazdy
WHERE model = 'meta';
```

  <div className="solution-check good">
    <strong>Dlaczego to przechodzi:</strong> są dwie funkcje agregujące i filtr na model <code>meta</code>.
  </div>
  <div className="solution-check bad">
    <strong>Za co bym obciął:</strong> inne aliasy niż w poleceniu. Egzaminator porównuje wynik z oczekiwanym widokiem.
  </div>
</div>

<div className="exam-solution-block">
  <div className="solution-head">
    <span>ZAPYTANIE 3</span>
    <h3>Pojazdy alfa z nazwą koloru i dopłatą</h3>
  </div>

```sql
SELECT marka, model, cena, nazwa, doplata
FROM pojazdy
INNER JOIN kolory ON pojazdy.kolor = kolory.id
WHERE model = 'alfa';
```

  <div className="solution-check good">
    <strong>Dlaczego to przechodzi:</strong> JOIN idzie po kluczu obcym, a wynik zawiera dane z obu tabel.
  </div>
  <div className="solution-check bad">
    <strong>Za co bym obciął:</strong> cartesian join bez <code>ON</code>, zły warunek łączenia albo pominięcie <code>doplata</code>.
  </div>
</div>

<div className="exam-solution-block">
  <div className="solution-head">
    <span>ZAPYTANIE 4</span>
    <h3>Dwa losowe pojazdy</h3>
  </div>

```sql
SELECT marka, model, cena
FROM pojazdy
ORDER BY RAND()
LIMIT 2;
```

  <div className="solution-check good">
    <strong>Dlaczego to przechodzi:</strong> <code>RAND()</code> miesza rekordy, <code>LIMIT 2</code> bierze dwa.
  </div>
  <div className="solution-check bad">
    <strong>Za co bym obciął:</strong> pobranie wszystkich aut i losowanie ręcznie bez sensu. Miało być prosto.
  </div>
</div>

### Połączenie z bazą

```php
<?php
$conn = new mysqli("localhost", "root", "", "samochody");
$conn->set_charset("utf8");
```

<div className="exam-teacher-note">
  <strong>Ocena:</strong> brak <code>set_charset("utf8")</code> to typowy błąd ucznia, który „u mnie działa”.
  Na egzaminie polskie znaki mają się wyświetlać poprawnie. Koniec dyskusji.
</div>

### Sekcja lewa — tabela pojazdów alfa

```php
$query = "SELECT marka, model, cena, nazwa, doplata
          FROM pojazdy
          INNER JOIN kolory ON pojazdy.kolor = kolory.id
          WHERE model = 'alfa'";
$result = $conn->query($query);

echo "<table>";
echo "<tr><th>Marka</th><th>Model</th><th>Kolor</th><th>Cena całkowita</th></tr>";

while ($row = $result->fetch_assoc()) {
    $cena_calkowita = $row['cena'] + $row['doplata'];
    echo "<tr>";
    echo "<td>" . $row['marka'] . "</td>";
    echo "<td>" . $row['model'] . "</td>";
    echo "<td>" . $row['nazwa'] . "</td>";
    echo "<td>" . $cena_calkowita . "</td>";
    echo "</tr>";
}

echo "</table>";
```

<div className="exam-grading-rubric">
  <div><strong>+ punkt</strong><span>JOIN na właściwych kolumnach</span></div>
  <div><strong>+ punkt</strong><span>pętla <code>while</code> przechodzi po wynikach</span></div>
  <div><strong>+ punkt</strong><span>cena całkowita liczona w PHP</span></div>
  <div><strong>- punkt</strong><span>liczenie ceny w SQL mimo polecenia</span></div>
</div>

### Sekcja środkowa — losowanie i rowspan

```php
$query = "SELECT marka, model, cena FROM pojazdy ORDER BY RAND() LIMIT 2";
$result = $conn->query($query);
$nr = 1;

while ($row = $result->fetch_assoc()) {
    echo "<img src='a{$nr}.jpg' alt='Pojazd {$nr}'>";
    echo "<table>";
    echo "<tr><td rowspan='2'>" . $row['cena'] . "</td>";
    echo "<td>Marka: " . $row['marka'] . "</td></tr>";
    echo "<tr><td>Model: " . $row['model'] . "</td></tr>";
    echo "</table>";
    $nr++;
}
```

<div className="exam-verdict danger">
  <div className="exam-verdict-label">tu uczniowie tracą punkty</div>
  <div className="exam-copy">
    <code>rowspan='2'</code> nie jest dekoracją. To wymaganie makiety. Jeśli tabela wygląda podobnie,
    ale nie ma <code>rowspan</code>, to w kodzie nie spełniasz polecenia.
  </div>
</div>

### Sekcja prawa i zamknięcie

```php
echo "<p>Telefon: 111 222 444</p>";
echo "<img src='a3.png' alt='Kontakt'>";

$conn->close();
```

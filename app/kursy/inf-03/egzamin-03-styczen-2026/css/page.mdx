## Kolejność stylowania

1. Najpierw ustaw główne sekcje strony: `header`, `nav`, `main`, `section`, `aside`, `footer`.
2. Potem dopasuj szerokości, wysokości, `display`, `float`, `flex` albo `grid` — zależnie od polecenia.
3. Na końcu dodaj detale: kolory, obramowania, marginesy, paddingi, hover i rozmiary obrazów.

<Callout type="warning" title="Najczęstszy błąd">

Uczniowie zaczynają od kolorków, a dopiero potem próbują naprawić układ. Na egzaminie to zła kolejność. Jeżeli layout jest rozjechany, nawet poprawny kolor nie uratuje punktów za strukturę strony.

</Callout>

## Co musi być zgodne z arkuszem

- Nazwy selektorów i plik CSS podpięty w HTML/PHP.
- Wymiary bloków, ustawienie obrazów i wyrównanie tekstu.
- Kolory zapisane tak, jak wymaga polecenie: nazwa koloru, HEX albo RGB.
- Zachowanie elementów interaktywnych, jeśli arkusz wymaga `:hover`, kursora albo zmiany tła.

## Szybki test przed oddaniem

- [ ] Strona nie wymaga przewijania poziomego na typowym ekranie.
- [ ] Obrazy nie wychodzą poza swoje kontenery.
- [ ] Formularz jest czytelny i przyciski są widoczne.
- [ ] Układ dalej wygląda poprawnie po odświeżeniu strony.
- [ ] CSS nie zawiera martwych selektorów, które nie pasują do HTML.

<Callout type="exam" title="Werdykt nauczyciela">
Najpierw dowody, potem kod. W arkuszu INF.03-02 egzaminator nie ocenia twojej intuicji, tylko zgodność z wymaganiami: nazwy tabel, kolumn, plików, zapytań i wyników.
</Callout>

## Co rozwiązuje ten arkusz

Portal dla sklepów z warzywami — baza `bazar`, strona PHP z nawigacją obrazkową, formularzem zamówień i obliczeniem wartości.

**Baza danych:** `bazar` — tabele: `towar` (id, nazwa, rodzaj, cena, plik), `sklep` (id, nazwa, adres), `zamowienie` (id, id_towar FK, id_sklep FK, kilogramy).

## Jak rozbić wymagania

1. Wypisz dane wejściowe: baza, tabele, obrazy, formularze i pliki wynikowe.
2. Oddziel rzeczy statyczne od dynamicznych. Statyczne robisz w HTML/CSS, dynamiczne w PHP albo JavaScript.
3. Każde zapytanie uruchom osobno przed wklejeniem do kodu. Jeśli SQL nie działa w bazie, nie zacznie działać magicznie w PHP.
4. Zachowaj nazwy z polecenia. Inna nazwa pliku, pola formularza albo tabeli to nie wariant — to błąd.

<Callout type="warning" title="Za co bym obciął">
Za pisanie zapytań 'na oko', bez sprawdzenia wyniku. Za brak klucza obcego, złą nazwę kolumny, dopisanie sortowania którego nie było w poleceniu albo pominięcie ograniczenia LIMIT/WHERE.
</Callout>

## Zapytania SQL — gotowe rozwiązania

Baza: `bazar`. Plik SQL: `baza-nowa.sql`.

### Zapytanie 1: 10 pierwszych towarów — nazwa i plik

```sql
SELECT nazwa, plik FROM towar LIMIT 10;
```

<Callout type="tip" title="Dlaczego nie ORDER BY?">
Polecenie mówi "10 pierwszych rekordów" — bez sortowania. `LIMIT 10` wystarczy. Nie dodawaj `ORDER BY` jeśli nie jest wymagany — egzaminator może uznać to za niezgodność z poleceniem.
</Callout>

### Zapytanie 2: identyfikator i nazwa towarów

```sql
SELECT id, nazwa FROM towar;
```

### Zapytanie 3: rodzaj, nazwa i cena towaru o id = 1

```sql
SELECT rodzaj, nazwa, cena FROM towar WHERE id = 1;
```

### Zapytanie 4: wstawienie rekordu do zamowienie

```sql
INSERT INTO zamowienie (id_towar, id_sklep, kilogramy)
VALUES (9, 2, 20);
```

<Callout type="exam">
Klucz główny jest nadawany automatycznie (AUTO_INCREMENT) — **nie wpisuj go** w INSERT. Egzaminator sprawdza, czy nie wymuszasz id ręcznie.
</Callout>

## Jak sprawdzić ten etap

- wynik każdego zapytania ma dokładnie te kolumny, których żąda polecenie;
- liczba rekordów zgadza się z warunkiem `LIMIT`, `WHERE` albo agregacją;
- w zapytaniach z więcej niż jedną tabelą jest jawny warunek połączenia;
- zapytanie modyfikujące dane nie wpisuje ręcznie wartości, które baza ma nadać automatycznie;
- zrzuty ekranów/import bazy robisz dopiero po sprawdzeniu wyniku, nie przed.

## O czym jest ten arkusz

Portal do diagnostyki chorób z relacją n:m (choroba ↔ objawy przez tabelę łączącą). Jedna strona zdrowie.php z listą chorób zakaźnych, listą rozwijaną i wyświetlaniem objawów w znacznikach span z pseudo-klasami nth-child.

**Baza danych:** `choroby` — tabele: `choroba` (id, nazwa, zakazna T/N), `objawy` (id, nazwa), `choroba_objawy` (id_choroba FK, id_objaw FK) — relacja n:m.

## Co musisz oddać — pliki

zdrowie.php: header (h1), nav (3 odnośniki external target=_blank), main z sekcją lewą (lista numerowana chorób zakaźnych — skrypt 1) i sekcją prawą (formularz z listą rozwijaną — skrypt 2, przycisk Sprawdź, div z objawami w span — skrypt 3), footer, obraz zdrowia.png poza blokami (position absolute).

## Grafika — co robić

zdrowia.png: tekst "ZDROWIA", tło przezroczyste, wysokość 100px, przycięte do szerokości tekstu.

## Zapytania SQL — gotowe rozwiązania

Baza: `choroby`. Plik SQL: `choroby.sql`.

### Zapytanie 1: choroby zakaźne posortowane

```sql
SELECT nazwa FROM choroba WHERE zakazna = 'T' ORDER BY nazwa ASC;
```

### Zapytanie 2: wszystkie choroby — id i nazwa

```sql
SELECT id, nazwa FROM choroba;
```

### Zapytanie 3: objawy choroby id=5 przez relację

```sql
SELECT o.nazwa FROM objawy o
JOIN choroba_objawy co ON o.id = co.id_objaw
WHERE co.id_choroba = 5;
```

### Zapytanie 4: choroby zakaźne + liczba objawów

```sql
SELECT c.nazwa, COUNT(*) AS liczba_objawow
FROM choroba c
JOIN choroba_objawy co ON c.id = co.id_choroba
WHERE c.zakazna = 'T'
GROUP BY c.id;
```

<Callout type="exam">
Zapytanie 4 łączy JOIN + WHERE + GROUP BY — trzy klauzule naraz na relacji n:m. To najtrudniejsze zapytanie w tym arkuszu. Przećwicz pisanie takich z pamięci.
</Callout>

## Skrypty PHP — rozwiązanie krok po kroku

### Połączenie z bazą

```php
<?php
$conn = mysqli_connect("localhost", "root", "", "choroby");
mysqli_query($conn, "SET CHARSET utf8");
```

### Skrypt 1 — lista chorób zakaźnych

```php
$wynik = mysqli_query($conn,
    "SELECT nazwa FROM choroba WHERE zakazna = 'T' ORDER BY nazwa ASC");
echo "<ol>";
while ($row = mysqli_fetch_array($wynik)) {
    echo "<li>" . $row['nazwa'] . "</li>";
}
echo "</ol>";
```

### Skrypt 2 — lista rozwijana wszystkich chorób

```php
echo "<select name='choroba'>";
$wynik2 = mysqli_query($conn, "SELECT id, nazwa FROM choroba");
while ($row = mysqli_fetch_array($wynik2)) {
    echo "<option value='" . $row['id'] . "'>" . $row['nazwa'] . "</option>";
}
echo "</select>";
```

### Skrypt 3 — objawy wybranej choroby w span

```php
if (isset($_POST['choroba'])) {
    $id_choroby = $_POST['choroba'];
    $wynik3 = mysqli_query($conn,
        "SELECT o.nazwa FROM objawy o
         JOIN choroba_objawy co ON o.id = co.id_objaw
         WHERE co.id_choroba = $id_choroby");

    echo "<div class='objawy'>";
    while ($row = mysqli_fetch_array($wynik3)) {
        echo "<span>" . $row['nazwa'] . "</span> ";
    }
    echo "</div>";
}
```

### Zamknięcie

```php
mysqli_close($conn);
?>
```

<Callout type="exam">
`span:nth-child(n)` — pseudo-klasa, która wybiera n-ty element potomny. Polecenie wymaga różnych rozmiarów czcionki dla 2., 4., 5., 7., 9. i 10. spanu. To tworzy efekt "chmury tagów" z objawami o różnych wielkościach.

## Wnioski z tego arkusza

1. **Relacja n:m** — tabela łącząca `choroba_objawy` to klasyczny wzorzec. Musisz umieć pisać JOIN przez tabelę łączącą.

2. **position: absolute** — obraz zdrowia.png jest pozycjonowany absolutnie (left: 100px, top: 500px). Bez `position: absolute` na img, obraz zostanie w normalnym przepływie dokumentu.

3. **nth-child na span** — tworzy efekt "chmury tagów". Egzaminator sprawdza, czy użyłeś pseudo-klasy (a nie klas CSS na poszczególnych span).

4. **target=_blank** — wszystkie odnośniki w nawigacji otwierają się w nowym oknie. Bez tego atrybutu tracisz punkt za "zawartość witryny".

5. **margin: 50px 180px 0 50px** — skrócony zapis marginesów (góra/prawo/dół/lewo). Na egzaminie szybciej niż pisanie 4 osobnych `margin-top/right/bottom/left`.

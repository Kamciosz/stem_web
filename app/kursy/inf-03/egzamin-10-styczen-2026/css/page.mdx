Najpierw selektory z polecenia, potem wygląd. Jeżeli polecenie wymaga selektora znacznika, klasy albo konkretnego wymiaru, nie zastępuj tego 'lepszym' rozwiązaniem.
</Callout>

## Kolejność pracy nad stylem

1. Ustaw globalne fonty i kolory tła.
2. Zbuduj układ głównych sekcji: header, nav, main, aside/section, footer.
3. Dopiero potem dopracuj obrazy, tabele, linki, formularze i paragrafy.
4. Na końcu porównaj z makietą: wysokości, szerokości, wyrównania, obramowania i overflow.

<Callout type="warning" title="Za co bym obciął">
Za `ładniej niż w makiecie`, za pominięcie `overflow`, za zły font, za brak wymaganej ramki, za użycie klasy tam, gdzie polecenie wymaga selektora znacznika, i za obraz w złym rozmiarze.
</Callout>

## Styl CSS (styl.css)

```css
* {
    font-family: Garamond;
    text-align: center;
}

main {
    display: flex;
    flex-wrap: wrap;
}

aside {
    background-color: #3A3C3E;
    color: white;
    width: 10%;
    height: 600px;
}

main > section:first-of-type {
    width: 45%;
    height: 600px;
    overflow: auto;
}

section:last-of-type {
    width: 45%;
    height: 600px;
    color: white;
    background-size: cover;
}

h1 {
    background-color: rgba(211, 211, 211, 0.3);
    padding: 20px 0;
    margin: 0;
    font-size: 2em;
}

h2 {
    font-size: 2em;
    font-variant: small-caps;
}

li {
    text-align: left;
}

a {
    color: Gainsboro;
    font-size: 120%;
    font-style: italic;
}

a:hover {
    color: white;
}
```

<Callout type="exam">
`background-size: cover` — obraz tła automatycznie dopasowuje się do rozmiaru bloku. `rgba(211,211,211,0.3)` — przezroczysty szary na h1 (opacity 0.3). `font-variant: small-caps` — kapitaliki. Te trzy właściwości razem tworzą profesjonalny layout bloga.
</Callout>

## Mini-rubryka CSS

Weryfikator nie czyta twoich intencji. Patrzy na efekt i wymagania:

- selektor jest taki, jak w poleceniu;
- wymiar nie jest przypadkowy, tylko zgodny z arkuszem;
- kolor, font i obramowanie są wpisane jawnie;
- obraz nie rozciąga się bez zachowania proporcji;
- układ działa po odświeżeniu, nie tylko po ręcznej zmianie w DevTools.

Jeżeli któryś punkt jest „prawie”, to jest do poprawy. Na INF.03 „prawie jak makieta” oznacza utratę punktów za szczegół, który był wprost opisany.

## Kontrola CSS przed przejściem dalej

- strona nie ma poziomego scrolla;
- główne bloki mają wysokości i szerokości zgodne z poleceniem;
- grafiki są w dobrych folderach i mają poprawne `src`;
- formularze, tabele i paragrafy dziedziczą style zgodnie z selektorami;
- po odświeżeniu strony wygląd nadal jest taki sam, nie tylko po ręcznej zmianie w DevTools.

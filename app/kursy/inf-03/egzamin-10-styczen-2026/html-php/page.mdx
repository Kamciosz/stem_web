## O czym jest ten arkusz

Blog kulinarny z relacją n:m (potrawy ↔ alergeny). Jedna strona przepisy.php z nawigacją odnośnikową (GET), 4 skryptami PHP, tłem sekcji ustawianym inline z bazy danych. Grafika separator.png do kadrowania.

**Baza danych:** `przepisy` — tabele: `potrawy` (id, nazwa, trudnosc, kalorie, przepis, plik), `rodzaje` (id, rodzaj), `alergeny` (id, nazwa), `lista_alergenow` (id_potrawy FK, id_alergenu FK) — relacja n:m.

## Co musisz oddać — pliki

przepisy.php: aside (10 odnośników GET + autor), main (h1 rodzaj — skrypt 1, h2 nazwa + trudność/kalorie — skrypt 2, separator.png, p alergeny — skrypt 3, h2 Składniki + lista, p przepis — skrypt 4), section (tło inline z bazy + h1 "Blog Kulinarny"). Grafika: separator.png skadrowany do pierwszego separatora.

## Zapytania SQL — gotowe rozwiązania

### Zapytanie 1: nazwa, trudność, kalorie potrawy id=7

```sql
SELECT nazwa, trudnosc, kalorie FROM potrawy WHERE id = 7;
```

### Zapytanie 2: nazwa i rodzaj potrawy id=7 (JOIN)

```sql
SELECT p.nazwa, r.rodzaj FROM potrawy p
JOIN rodzaje r ON p.id_rodzaj = r.id
WHERE p.id = 7;
```

### Zapytanie 3: nazwa i alergeny potrawy id=7 (JOIN n:m)

```sql
SELECT p.nazwa, a.nazwa AS alergen
FROM potrawy p
JOIN lista_alergenow la ON p.id = la.id_potrawy
JOIN alergeny a ON la.id_alergenu = a.id
WHERE p.id = 7;
```

<Callout type="exam">
Zapytanie 3 łączy TRZY tabele przez relację n:m — to najtrudniejsze zapytanie w tym arkuszu. Musisz przejść: potrawy → lista_alergenow → alergeny. Przećwicz ten wzorzec.
</Callout>

### Zapytanie 4: przepis i plik potrawy id=7

```sql
SELECT przepis, plik FROM potrawy WHERE id = 7;
```

## Skrypty PHP — rozwiązanie krok po kroku

### Połączenie z bazą + inicjalizacja ID

```php
<?php
$conn = mysqli_connect("localhost", "root", "", "przepisy");
mysqli_query($conn, "SET CHARSET utf8");

$id = isset($_GET['id']) ? $_GET['id'] : 7;
```

<Callout type="tip" title="Domyślna wartość ID">
Polecenie mówi: "jeśli nie wysłano GET, zmienna = 7". To typowy wzorzec — strona pokazuje domyślną potrawę gdy użytkownik jeszcze nic nie wybrał.
</Callout>

### Skrypt 1 — rodzaj potrawy (nagłówek h1)

```php
$wynik1 = mysqli_query($conn,
    "SELECT r.rodzaj FROM potrawy p
     JOIN rodzaje r ON p.id_rodzaj = r.id WHERE p.id = $id");
$row = mysqli_fetch_array($wynik1);
echo "<h1>" . $row['rodzaj'] . "</h1>";
```

### Skrypt 2 — nazwa, trudność, kalorie

```php
$wynik2 = mysqli_query($conn,
    "SELECT nazwa, trudnosc, kalorie FROM potrawy WHERE id = $id");
$row = mysqli_fetch_array($wynik2);
echo "<h2>" . $row['nazwa'] . "</h2>";

$trudnosc_tekst = ["", "łatwe", "średnie", "trudne"];
echo "<p>Trudność: " . $trudnosc_tekst[$row['trudnosc']]
     . ", Kalorie: " . $row['kalorie'] . "</p>";
```

### Skrypt 3 — alergeny

```php
$wynik3 = mysqli_query($conn,
    "SELECT a.nazwa FROM potrawy p
     JOIN lista_alergenow la ON p.id = la.id_potrawy
     JOIN alergeny a ON la.id_alergenu = a.id
     WHERE p.id = $id");
echo "Alergeny: ";
while ($row = mysqli_fetch_array($wynik3)) {
    echo $row['nazwa'] . " ";
}
```

### Skrypt 4 — przepis + plik do tła sekcji

```php
$wynik4 = mysqli_query($conn,
    "SELECT przepis, plik FROM potrawy WHERE id = $id");
$row = mysqli_fetch_array($wynik4);
echo "<p>" . $row['przepis'] . "</p>";
$plik_tla = $row['plik'];
```

Sekcja z tłem inline (w HTML):

```php
echo "<section style=\"background-image: url('$plik_tla'); background-size: cover;\">";
echo "<h1>Blog Kulinarny</h1>";
echo "</section>";
```

### Zamknięcie

```php
mysqli_close($conn);
?>
```

## Wnioski z tego arkusza

1. **4 skrypty w jednym pliku** — więcej niż zwykle. Każdy odpowiada za inną sekcję strony. Ważna organizacja kodu.

2. **Tło sekcji z bazy inline** — pole `plik` z bazy staje się `background-image` w atrybucie `style`. To łączy PHP (dane z bazy) z CSS (inline style).

3. **Relacja n:m przez 3 tabele** — potrawy → lista_alergenow → alergeny. Podwójny JOIN. Najtrudniejsze zapytanie sesji.

4. **Trudność jako liczba → tekst** — baza ma `trudnosc=1/2/3`, ale strona wyświetla "łatwe/średnie/trudne". Konwersja w PHP.

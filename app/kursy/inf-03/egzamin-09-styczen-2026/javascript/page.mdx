## Mapa technologii INF.03 dla tego arkusza

INF.03 obejmuje tworzenie stron i aplikacji internetowych oraz baz danych. Dlatego w tym kursie rozbijamy front-end na konkretne technologie zamiast trzymać wszystko w jednym dziale. Ucz się etapami:

- **HTML** — struktura dokumentu, nazwy plików, formularze, linki, grafika i semantyczne bloki.
- **PHP** — połączenie z bazą, mysqli/charset, pętle po wyniku, GET/POST, obliczenia lub zapis danych.
- **JavaScript** — zdarzenia użytkownika, DOM, obliczenia po stronie klienta, createElement/appendChild albo przełączanie widoku.

<Callout type="exam" title="Dlaczego osobno HTML, PHP i JavaScript?">
HTML opisuje strukturę strony, CSS wygląd, PHP działa po stronie serwera i zwykle łączy się z bazą, a JavaScript działa w przeglądarce. Na egzaminie punkt jest za wymaganie, nie za nazwę działu — dlatego technologię trzeba rozpoznać po tym, gdzie kod się wykonuje.
</Callout>

1. **Wejście** — skąd biorą się dane: baza, formularz, pliki graficzne albo zdarzenie JavaScript.

3. **Wyjście** — co ma zobaczyć egzaminator: tabela, lista, komunikat, obraz albo zmieniony fragment strony.

## JavaScript — co jest oceniane

JS w tych arkuszach działa w przeglądarce: obsługuje kliknięcia lub `onmouseover`, zmienia DOM, liczy wynik albo dodaje obraz. Używaj `const` dla wartości nieprzypisywanych ponownie i `let` dla liczników pętli. Nie używaj starego deklarowania zmiennych w nowych rozwiązaniach.

## Minimum do zapamiętania z JS

- Elementy DOM pobieraj przez `const`, np. `const wynik = document.getElementById("wynik")`.
- Liczniki pętli zapisuj jako `let`, np. `for (let i = 1; i <= 10; i++)`.
- `input[type=file].value` może zwracać `C:\fakepath\plik.png` — wytnij nazwę pliku przed użyciem.
- `element.src` zwykle jest pełnym URL-em, więc przy nazwach plików używaj `includes()`.

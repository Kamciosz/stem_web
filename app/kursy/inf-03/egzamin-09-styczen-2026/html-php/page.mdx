## Zapytania SQL — gotowe rozwiązania

### Zapytanie 1: ważni klienci

```sql
SELECT imie, kolor, cena FROM klienci WHERE topKlient = 1;
```

### Zapytanie 2: klienci z dnia 2024-05-23

```sql
SELECT imie, cena FROM klienci WHERE data = '2024-05-23';
```

### Zapytanie 3: klienci z kształtem migdał (JOIN)

```sql
SELECT k.imie, k.kolor, ks.nazwa
FROM klienci k
JOIN ksztalty ks ON k.ksztalt = ks.id
WHERE ks.nazwa = 'migdał';
```

### Zapytanie 4: dodanie kolumny doplata

```sql
ALTER TABLE wzory ADD COLUMN doplata INT;
```

<Callout type="exam">
`ALTER TABLE ... ADD COLUMN` — to jedyny arkusz w tej sesji, który wymaga modyfikacji struktury tabeli (DDL zamiast DML). Musisz znać różnicę między SELECT/INSERT (DML) a ALTER/CREATE (DDL).
</Callout>

## Skrypt 2 — przełączanie sekcji (JavaScript onmouseover)

```javascript
function pokaz(nr) {
    var sekcje = document.querySelectorAll("section");
    var przyciski = document.querySelectorAll("nav button");

    for (var i = 0; i < sekcje.length; i++) {
        sekcje[i].style.display = "none";
        przyciski[i].style.backgroundColor = "Crimson";
    }

    sekcje[nr].style.display = "block";
    przyciski[nr].style.backgroundColor = "Salmon";
}
```

HTML przycisków w nav:

```html
<button onmouseover="pokaz(0)">Kolor</button>
<button onmouseover="pokaz(1)">Kształt</button>
<button onmouseover="pokaz(2)">Wzór</button>
```

<Callout type="warning" title="onmouseover, nie onclick">
Polecenie mówi "zdarzenie najechania kursorem" — to `onmouseover`. Użycie `onclick` = utrata punktu za "działanie witryny".
</Callout>

## Weryfikator w złym humorze

Sprawdzałbym tę pracę tak:

- czy pliki mają nazwy z polecenia, a nie twoje własne;
- czy formularz wysyła dane właściwą metodą i do właściwego pliku;
- czy PHP/JS uruchamia się dopiero wtedy, kiedy ma dane wejściowe;
- czy wynik jest widoczny na stronie bez warningów w przeglądarce i bez błędów PHP;
- czy kod realizuje dokładnie polecenie, a nie 'podobny efekt'.

To jest miejsce, gdzie odpadają punkty za drobiazgi. Drobiazg w arkuszu to nadal wymaganie.

## Wnioski z tego arkusza

1. **Arkusz mieszany PHP+JS** — skrypt 1 można napisać w PHP lub JS (polecenie daje wybór). Skrypt 2 MUSI być w JS (działa po stronie klienta).

2. **onmouseover zamiast onclick** — najczęstszy błąd. Czytaj polecenie dosłownie.

3. **display: none/block** — przełączanie widoczności sekcji. Domyślnie sekcja1 widoczna (block), reszta ukryta (none). Skrypt 2 odwraca to.

4. **ALTER TABLE** — jedyny arkusz z DDL. Reszta arkuszy ma tylko SELECT/INSERT.

5. **border-radius: 100%** — tworzy okrągłe miniaturki wzorów (klasa .wzory). Na prostokątnych obrazach daje elipsy.
